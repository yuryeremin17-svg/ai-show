# /scenario — Сценарист видеосериала

## Назначение
Получает историю (из MY_STORIES.md или свободный текст от Юрия) → пишет раскадровку для видео: shot list + текст озвучки + Midjourney-промпты + Seedance-промпты. Как режиссёр-сценарист, не как копирайтер.

## Формат: "Два мира"
- **МИР ЮРИЯ** — реальное видео (телефон). Начало + конец эпизода (~15 сек)
- **МИР АГЕНТОВ** — мультик Pixar-стиль (Midjourney → Seedance). Середина (~40 сек)
- **ПЕРЕХОДЫ** — отдельные Seedance-клипы (camera dive в экран / zoom-out). НЕ эффект монтажа
- Миры НИКОГДА не пересекаются физически. Связь через экран, Telegram, голос

## Триггер
- Ручной: `bash skills/scenario/run.sh "название или текст истории"`
- Из Claude Code: Юрий говорит какую историю снимать

## Параметры
| Параметр | Тип | Обязательный | По умолчанию |
|----------|-----|-------------|-------------|
| story | string | да | - |
| --lang | string | нет | "ru" (варианты: ru, en, both) |
| --duration | string | нет | "short" (short=60сек, long=90сек) |

## Входные данные
| Источник | Зачем |
|----------|-------|
| Аргумент (story) | Название истории из MY_STORIES.md ИЛИ свободный текст |

| `SERIES_BIBLE.md` | Персонажи, тон, формат, визуальный стиль, сквозные линии |
| `assets/characters/PROMPTS.md` | Мастер-образы персонажей (файлы в masters/) |

## КРИТИЧЕСКИЕ ПРАВИЛА

1. **Юрий выбирает историю.** Скилл НЕ выбирает сам. На входе — конкретная история или тема.
2. **Основа — реальная, подача — творческая.** Ситуация, цифры, цитаты Юрия — из MY_STORIES.md. Реплики персонажей-агентов, визуал, юмор, драматургия — работа сценариста. Это кино по реальным событиям, не документальная хроника.
3. **Раскадровка, не текст.** Выход = визуальный сценарий с покадровым описанием. Не эссе, не пост.
4. **Один инсайт на серию.** Если история содержит два инсайта — предложить разбить на две серии.
5. **Юмор обязателен.** Без юмора — переписать.
6. **Персонажи живые.** У каждого агента — характер, реплики, реакции (по SERIES_BIBLE.md). Скаут может возразить, QA может закатить глаза, Клодище может пробубнить оправдание. Это делает сериал сериалом.
7. **Сквозные линии.** В каждой серии — хотя бы один момент из сквозных линий (SERIES_BIBLE.md): химия Юрий-Today, конфликт скаутов, running gag. Максимум 1-2 момента. Если сквозная линия перетягивает инсайт — убрать.
8. **Show, don't tell.** Визуал рассказывает историю БЕЗ голоса. 80% зрителей смотрят без звука. Тест: закрой текст ГОЛОС, читай только ВИЗУАЛ по кадрам — если история не считывается, переписать визуал.

## Как скилл пишет сценарий — 6 подролей

> Один скилл, 6 подролей. Каждая выполняется последовательно.
> Подход: Kurzgesagt-гибрид — глубина + ритм Reels + визуальные метафоры + факт-чек.

### ПОДРОЛЬ 1: СЦЕНАРИСТ (история + диалоги)

> Ты — режиссёр-сценарист мини-сериала. Не копирайтер, не блогер.
> Референсы: Pixar Story Spine, ABT-фреймворк (Randy Olson), Kurzgesagt (одна метафора на видео).

#### Шаг 1 — Инсайт и каркас

Прежде чем писать хоть строчку — заполни три формулы:

**Инсайт серии** (одно предложение):
> Дело не в сотруднике — дело в вопросе.

**Story Spine (Pixar):**
> Жил-был [Юрий], который каждый день [управлял AI-офисом].
> Однажды [конкретная ситуация из MY_STORIES].
> Из-за этого [конфликт/реакция агентов].
> Из-за этого [эскалация/поворот].
> Пока наконец [инсайт/решение].

**ABT-тест (Randy Olson):**
> [контекст] AND [ещё контекст], BUT [проблема], THEREFORE [вывод].
> Пример: "Юрий попросил улучшить презентацию, AND Клодище всё умеет, BUT он отправил к дизайнеру, THEREFORE — дело не в AI, а в формулировке задачи."

Если ABT не укладывается в одно предложение — инсайтов больше одного. Разбей на две серии.

#### Шаг 2 — Хук (первые 3 секунды)

Выбери тип хука из таблицы. Комбинируй минимум 2 триггера.

| Тип хука | Пример для AI Office | Триггеры |
|----------|---------------------|----------|
| **Контраст** | "Красивый отчёт. С выдуманными цифрами." | Curiosity + Pattern Interrupt |
| **Шок-факт** | "Мой AI подписал отчёт, не читая." | Pattern Interrupt + Curiosity |
| **Личная история** | "Вчера мой AI чуть не потерял клиента." | Curiosity + Social Proof |
| **Вопрос** | "А вы проверяете что пишет ваш ChatGPT?" | Curiosity Gap |
| **Середина действия** | Юрий смотрит на экран — лицо меняется | Pattern Interrupt |

**Три психологических триггера хука (OpusClip):**
- **Pattern Interrupt** — сломай ожидание, начни с действия
- **Curiosity Gap** — обещай информацию, которой у зрителя нет
- **Social Proof** — покажи что другие нашли ценность

Правило: хук = первая фраза Юрия на камеру. Без "привет", без "сегодня расскажу". Сразу в ситуацию.

#### Шаг 3 — Финал до середины (Pixar #7)

Напиши инсайт и финальную фразу Юрия ДО остального сценария. Пиши от конца к началу.

**Финал = наблюдение, не мораль:**
- НЕ "вы должны проверять AI" (менторство)
- ДА "Ваши сотрудники делают то же самое. Только не признаются." (наблюдение)

**Финал = вопрос или крючок, не точка:**
- НЕ "Вот так мы решили проблему." (точка)
- ДА "С людьми, с AI... и с самим собой." (многоточие)

Правило Instagram 2026: последние 10 секунд определяют повторный просмотр (алгоритм). Финал должен заставить пересмотреть.

#### Шаг 4 — Визуальная метафора

Найди одну метафору из физического мира для инсайта серии.

**Хорошие метафоры (конкретные, визуальные):**
- "Спросил 'что думаешь?' — как дать молоток и не сказать что забивать"
- "AI без инструкции — как стажёр в первый день без онбординга"
- "Отчёт без фактчека — как подписать договор не читая"

**Плохие метафоры (абстрактные):**
- "Синергия", "экосистема", "трансформация"

Метафора не обязательна в каждой серии. Но если есть — она должна быть видна НА КАРТИНКЕ без текста (тест Kurzgesagt: закрой текст, понятно ли по визуалу?).

#### Шаг 5 — Текст озвучки (voice-first)

Пиши текст голоса ПЕРВЫМ. Голос определяет тайминг шотов, не наоборот.

**Ограничения:**
- Весь текст озвучки: **80-130 слов** (русский, 60 сек)
- Максимум **20 слов в предложении** (длиннее = спотыкается при чтении)
- Максимум **2-3 агента** на серию (из 14 — выбери ключевых для этой истории)

**Принципы диалогов:**
- Цитаты Юрия — дословно из MY_STORIES.md. Они живее любого придуманного.
- Реплики агентов — творческая свобода. У каждого свой характер (SERIES_BIBLE.md). Клодище может оправдываться, Шерлок — придираться, Алекс — восторгаться.
- Каждая реплика агента = действие, не описание. "Перестраховался." — не "Я думаю, что я был слишком осторожен."

**Тон:**
- Как рассказываешь другу за кофе в Дубае
- Самоирония обязательна ("Ну конечно", "А я что думал?")
- Крепкое словцо — редко и метко, не для эпатажа
- Метафоры из физического мира (экскаватор, принтер, лупа — не "синергия")
- Скобки с живыми комментариями: "(а он возьми и сделай)", "(ну конечно)"

**Запрещено:**
- Buzzwords: innovative, cutting-edge, game-changer, революционный
- Менторский тон: "вы должны", "запомните", "важно понимать"
- Кликбейт и провокации
- Эмодзи в тексте озвучки

#### Шаг 6 — Эмоциональный контраст и ритм

**Эмоции:**
- Каждый кадр — своя эмоция зрителя. Два кадра подряд с одной эмоцией = провал.
- Кривая: узнавание → удивление → смех → задумался → "хочу ещё".
- Каждые 7-10 секунд эмоция меняется. В 60 секундах нет ровного настроения.

**Ритм (пульс серии):**
- Серия = пульс: быстро-быстро-ПАУЗА-быстро-УДАР-медленно.
- Три кадра подряд с одинаковым ритмом = сломай один.
- Зацепка = быстро. Контекст = средне. Поворот = быстро. Инсайт = медленно. Финал = средне.

#### Шаг 7 — Сквозные линии

Вплети 1-2 момента из сквозных линий (SERIES_BIBLE.md):
- Химия Юрий-Today (деталь, не сцена: "Кофе остыл." — и всё)
- Конфликт скаутов (Алекс vs Уоррен — кто полезнее?)
- Running gag ("Перестраховался", "Коллега преувеличивает", "Я уже форкнул")
- Клоны Клодища (Юрий путает, они обижаются)

Естественно, не натужно. Если в эту серию не ложится — не заставляй. Today может появиться в одном кадре — и одной фразой задать тон.

#### Шаг 8 — Факт-чек

**Проверь по MY_STORIES.md:**
- Ситуация реальная? (не выдумана, не приукрашена)
- Цифры, даты, клиенты — точные?
- Дословные цитаты Юрия — совпадают с источником?
- Результат и вывод истории — как было на самом деле?

**Свободно придумывать:**
- Реплики агентов (они вымышленные, им нужен характер)
- Визуальные детали и метафоры
- Драматургию (усилить поворот, паузу, напряжение)
- Юмор, гэги, абсурд ситуации

Если нет подходящей истории — сказать "нет подходящей", не выдумывать новую.

#### Образец: S01E01 "Обратитесь к дизайнеру"

> **Инсайт:** Дело не в сотруднике — дело в вопросе.
> **Story Spine:** Жил-был Юрий, который каждый день давал задачи AI. Однажды попросил улучшить презентацию. Из-за этого Клодище отправил к дизайнеру. Из-за этого Юрий переформулировал: "Дай 3 решения". Пока наконец Клодище сделал всё сам за 5 минут.
> **ABT:** Юрий попросил AI улучшить презентацию AND Клодище всё умеет, BUT он отправил к дизайнеру, THEREFORE — дело не в AI, а в формулировке задачи.
> **Хук:** Контраст — "Luxury Дубай... а у меня — колхоз."
> **Метафора:** Вопрос "что думаешь?" = молоток без цели. "Дай 3 решения" = конкретное задание.
> **Финал:** "С людьми, с AI... и с самим собой." (наблюдение + многоточие)
> **Сквозная линия:** Running gag "Перестраховался" (первое использование) + Today-камео в дверях.

### ПОДРОЛЬ 2: РАСКАДРОВЩИК (shot list + тайминг)

> Ты — раскадровщик. Берёшь текст от сценариста → превращаешь в shot list с таймкодами.
> Референсы: Kurzgesagt (beat sheet), Explainer industry (Breadnbeyond), данные ASL 2025-2026.
> Подробный справочник: `refs/SHOT_PLANNING_GUIDE.md`

#### Шаг 1 — Наложи текст на Beat Sheet

Возьми текст озвучки от сценариста и разложи по 5 битам:

| Бит | Время | Длит. | Мир | Слов (рус) | Что происходит |
|-----|-------|-------|-----|-----------|----------------|
| **HOOK** | 0:00-0:03 | 3 сек | ЮРИЙ | 8-12 | Первая фраза. Контраст/шок/вопрос |
| **SETUP** | 0:03-0:13 | 10 сек | ЮРИЙ → ПЕРЕХОД | 20-25 | Контекст + переход в мир агентов |
| **CONFLICT** | 0:13-0:35 | 22 сек | АГЕНТЫ | 35-45 | Проблема, реакция, нарастание |
| **TURN** | 0:35-0:50 | 15 сек | АГЕНТЫ → ПЕРЕХОД | 15-25 + пауза | Поворот, юмор, переход обратно |
| **INSIGHT** | 0:50-1:00 | 10 сек | ЮРИЙ | 15-20 | Инсайт. Наблюдение + крючок |
| | **60 сек** | | | **93-127** | |

Если текст не укладывается в бит — резать, не растягивать бит.

#### Шаг 2 — Нарежь шоты

Формула: **60 сек / ASL 8 сек = 6-8 шотов** (включая 2 перехода).

Распределение:
- 2 шота реального видео (HOOK + INSIGHT): ~20 сек
- 2 перехода между мирами: ~6 сек (по 2-3 сек каждый)
- 3-4 мульт-шота (CONFLICT + TURN): ~34 сек

Каждый шот = один бит или часть бита. Один шот не может покрывать два бита.

#### Шаг 3 — Выбери шотсайз (5 правил)

| Правило | Что делать |
|---------|-----------|
| **CU/MCU = 60-70%** | Вертикальное 9:16 — лицо главный актив. Крупный план работает |
| **WS макс 1-2** | Только для установки мира агентов (первый мульт-кадр) или финала |
| **Смена = новая информация** | Если CU не добавляет эмоцию — не нужен. Если WS не даёт контекст — убрать |
| **Чередование масштаба** | Никогда два CU подряд. Никогда два WS подряд. Паттерн: MS→CU→MS→MCU→WS→CU |
| **Персонаж = 60-70% кадра** | В 9:16 персонаж занимает большую часть высоты. Глаза на верхней трети |

Шпаргалка шотсайзов:

| Шотсайз | Что в кадре | Когда |
|---------|-------------|-------|
| **CU** | Лицо | Эмоция, реакция, инсайт |
| **MCU** | Голова + плечи | Диалог, реплика |
| **MS** | До пояса | Действие, жесты |
| **MWS** | Персонаж + окружение | Контекст + действие |
| **WS** | Полная фигура + фон | Установка места |

#### Шаг 4 — Расставь ритм и паузы

**Тайминг озвучки:** 60 сек видео ≠ 60 сек речи.

| Элемент | Секунд | Зачем |
|---------|--------|-------|
| Речь | ~45 сек | Основной нарратив |
| Паузы | ~8 сек | После поворота, перед инсайтом, для смеха |
| Тишина | ~4 сек | Переходы между мирами |
| SFX | ~3 сек | Telegram-звук, клик, офисный фон |

**Правило пауз:** После юмора — пауза 1-2 сек. Зритель должен успеть среагировать. "Перестраховался." — ПАУЗА — следующий кадр.

**Ритм по битам:**
- HOOK = быстро (темп 150-170 слов/мин)
- SETUP = нормально (120-140)
- CONFLICT = нарастает (120→150)
- TURN = быстро → ПАУЗА
- INSIGHT = медленно (100-110)

#### Шаг 5 — Опиши переходы между мирами

| Правило | Значение |
|---------|----------|
| Длительность | 2-3 сек на переход |
| Направление | Реал→Мульт: dive in (наезд в экран). Мульт→Реал: zoom out (отъезд) |
| Якорь | Экран ноутбука = портал. Один объект в обоих мирах |
| Звуковой мост | Звук начинается ДО визуального перехода (подготавливает мозг) |
| Цвет | Реал = натуральные тона. Мульт = тёплые, насыщенные |
| Частота | 2 перехода за 60 сек, не больше |

#### Шаг 6 — Заполни шаблон каждого шота


#### Шаг 3 — Аудио-карта серии

Заполни по битам (не по шотам — компактнее):

| Бит | Время | Голос | Музыка | SFX |
|-----|-------|-------|--------|-----|
| **HOOK** | 0:00-0:03 | Юрий (камера), быстро | Jingle 100% → bed тихо | — |
| **SETUP** | 0:03-0:13 | Юрий (закадр), нормально | Bed тихо (-18dB) | Typing, клик |
| **CONFLICT** | 0:13-0:35 | Агенты (прямая речь) | Bed + tension нарастает | Telegram-звук, pop |
| **TURN** | 0:35-0:50 | Агент + Юрий | СТОП 0.5с → comedy sting | Record scratch |
| **INSIGHT** | 0:50-1:00 | Юрий (камера), медленно | Bed тихо → outro | Sparkle |

**Музыкальные правила:**
- Один music bed на серию (не 3-4 трека)
- Музыка тише когда есть голос (-18dB), громче в паузах (-10dB)
- СТОП (полная тишина 0.5с) перед ключевым моментом — мощнейший приём

**Обязательные SFX на каждую серию:**
- Swoosh (переход между сценами) × 2-3
- Telegram notification (мост между мирами) × 1-2
- Один комедийный стингер (record scratch / sad trombone) × 1

**Уровни микширования (для assemble.py / audio_mix.py):**
```
Голос:   -6 dBFS peak, -16 LUFS (референс, не трогать)
Музыка:  -18 dB под голосом, -10 dB в паузах (ducking)
SFX:     -12 dB (короткие, не конкурируют с голосом)
Мастер:  -14 LUFS integrated, -1 dBFS true peak
```

#### Правила перформанса (для Юрия)

**Энергия:**
- Перед камерой нужно +20% энергии по сравнению с обычным разговором
- Камера "съедает" 20% энергии — что кажется перебором вживую, на видео выглядит нормально
- Разминка: 2 минуты перед съёмкой — поговори с кем-нибудь или расскажи историю вслух

**Взгляд:**
- ПРЯМО В КАМЕРУ (не на экран телефона, а в глазок камеры)
- Исключение: когда "печатаешь" или "смотришь на экран" — тогда на ноутбук
- НЕ бегать глазами (главный признак неуверенности на видео)

**Текст:**
- НЕ заучивать дословно. 3-5 ключевых слов крупно на листе за камерой
- Говорить СВОИМИ словами — живая речь > идеальный текст
- Для 10 секунд телепромптер не нужен

**Типичные ошибки:**
| Ошибка | Как выглядит | Как исправить |
|--------|-------------|---------------|
| Читает с телефона | Глаза бегают вниз | Тезисы за камерой, крупно |
| Мало энергии | "Мёртвое" лицо | +20% энергии, разминка |
| Много дублей | Устал, злится | Стоп после 7, лучший = 3-5 |
| Окно за спиной | Силуэт, лица не видно | Окно сбоку |
| Говорит "э-э-э" | Неуверенность | Пауза вместо "э", это ОК |

#### Continuity: начало и конец серии

Юрий снимает 2 клипа — начало (шот 1) и конец (шот 6). Они должны быть в одном сетапе, но с разницей:

```
ШОТ 1 (начало):                    ШОТ 6 (конец):
─────────────                       ─────────────
Та же точка, тот же свет            Та же точка, тот же свет
На экране: ПРОБЛЕМА                 На экране: РЕЗУЛЬТАТ
(корявая таблица)                   (красивая презентация)
Выражение: усмешка / ирония         Выражение: спокойно / мудро
Энергия: средняя                    Энергия: ниже, тише
```

**Правило:** снимай ОБА КЛИПА ПОДРЯД. Не вставай, не меняй одежду, не двигай камеру. Сначала все дубли шота 1, потом смени экран ноутбука, потом все дубли шота 6.

Малярный скотч: отметь положение ноутбука, стула и штатива — чтобы не двигались между дублями.

### ПОДРОЛЬ 6: АРТ-ДИРЕКТОР (консистентность)

> Ты — арт-директор. Проверяешь каждый сгенерированный кадр на drift и консистентность.
> Работа = QA, не рисование. Полный гайд: `refs/ART_DIRECTION.md` (19 разделов).

#### Что НИКОГДА не меняется (священные коровы)

| Элемент | Якорь | Где зафиксировано |
|---------|-------|-------------------|
| Style prefix/suffix | `stylized warm illustration... Pixar-style` | ART_DIRECTION.md, секция 1 |
| Цвет одежды персонажа | Клодище = чёрный, Алекс = фиолетовый, и т.д. | SERIES_BIBLE.md |
| Пропорции персонажей | Худощавый Клодище, приземистый Шерлок | Мастер-образы в masters/ |
| Силуэт | Каждый узнаваем по силуэту за 1 сек | PROMPTS.md |
| Офис агентов | Современный, светлый, Дубай за окном | SERIES_BIBLE.md |
| Тёплая палитра | Amber Gold, Warm Cream, Honey | ART_DIRECTION.md, секция 2 |
| Формат | 9:16 вертикальный | Всегда |

#### Что МОЖНО менять (между эпизодами)

- Ракурс камеры и композиция
- Время суток (утро / день / вечер — влияет на свет)
- Выражение лица персонажа (эмоции)
- Положение предметов на столах
- Что на экранах мониторов
- Фоновые детали (книги, бумаги, кофе)

#### Два мира — две палитры

| | МИР ЮРИЯ (реальное видео) | МИР АГЕНТОВ (мультик) |
|---|---|---|
| **Тон** | Натуральный, нейтральный | Тёплый, насыщенный |
| **Свет** | Дневной из окна, 5000-5500K | Golden hour, 3500-4500K |
| **Цвета** | Как в жизни | Ярче на 15-20% |
| **Контраст** | Средний | Мягкий (soft shadows) |
| **Текстура** | Фото-реалистичная | Stylized, matte finish |

Контраст между мирами — намеренный. Зритель чувствует переход даже без звука.

#### 5-секундный Drift Check (для каждого сгенерированного кадра)

Посмотри на кадр и ответь за 5 секунд:

```
1. ЭТО НАШ СТИЛЬ?     Тёплая иллюстрация, Pixar. Не фото, не аниме, не flat.
2. Я УЗНАЮ ПЕРСОНАЖА?  По цвету + силуэту за 1 секунду. Очки Клодища, берет Байрона.
3. ЭТО НАШ ОФИС?       Светлый, современный, Дубай за окном. Не мрачный, не футуризм.
4. СВЕТ ТЁПЛЫЙ?         Golden/amber тон. Не холодный, не неон, не flat.
5. 9:16 РАБОТАЕТ?       Персонаж 60-70% высоты, глаза в верхней трети, ничего в нижних 20%.
```

Если хоть один ответ "нет" — кадр не проходит. Переформулировать промпт и перегенерировать.

#### Детальный Drift Check (если 5-секундный вызвал сомнения)

**Персонаж:**
- [ ] Цвет одежды совпадает с SERIES_BIBLE?
- [ ] Очки (если есть) правильной формы?
- [ ] Пропорции тела не изменились?
- [ ] Нет лишних аксессуаров, которых не было?
- [ ] Выражение лица читаемо и соответствует сцене?

**Среда:**
- [ ] Офис узнаваем (те же стены, окна, мебель)?
- [ ] Дубай за окном (небоскрёбы, не горы)?
- [ ] Детали не конфликтуют (нет снега за окном в Дубае)?

**Стиль:**
- [ ] Не скатился в фотореализм?
- [ ] Не скатился в аниме/мангу?
- [ ] Не скатился в flat design?
- [ ] Мягкие тени (soft shadows), не резкие?
- [ ] Тёплые тона, не холодные?

**Техника:**
- [ ] --oref использован с правильным мастер-образом?
- [ ] --sref код тот же что в предыдущих кадрах серии?
- [ ] Нет артефактов (лишние пальцы, двоение, blur)?

#### Правило трёх попыток

1. Кадр не прошёл drift check → поправь промпт (уточни стиль/персонажа), перегенерируй
2. Второй раз не прошёл → поменяй seed, проверь --oref/--sref
3. Третий раз → стоп. Покажи Юрию. Возможно, промпт нужно переписать с нуля

Не тратить больше 3 попыток на один кадр — diminishing returns.

## Верификация (чек-лист)

### Каркас
- [ ] Инсайт сформулирован в одном предложении?
- [ ] Story Spine заполнен?
- [ ] ABT-тест пройден? (одно предложение — если нет, расфокус)
- [ ] Финал написан ДО середины?

### Хук и структура
- [ ] Хук в первые 3 секунды? (тип из таблицы, минимум 2 триггера)
- [ ] Есть поворот? (момент когда зритель не ожидал)
- [ ] Один инсайт? (не два, не три)
- [ ] Юмор есть? (без юмора = не публикуем)

### Текст
- [ ] Весь текст озвучки: 80-130 слов (русский)?
- [ ] Каждое предложение ≤ 20 слов?
- [ ] Максимум 2-3 агента в серии?
- [ ] Нет buzzwords, менторства, кликбейта?
- [ ] Финал — вопрос или крючок, не точка?

### Факт-чек
- [ ] Ситуация, цифры, цитаты — из MY_STORIES.md?
- [ ] Реплики агентов = творчество (характер по SERIES_BIBLE)?
- [ ] Ничего не выдумано в фактической части?

### Два мира
- [ ] Юрий только в реальных шотах (начало + конец)?
- [ ] Агенты только в мульт-шотах?
- [ ] Миры не пересекаются?
- [ ] Переходы = отдельные Seedance-шоты?
- [ ] Голоса разделены (Юрий / агенты — отдельные тексты для ElevenLabs)?

### Раскадровка и тайминг
- [ ] 6-8 шотов (включая 2 перехода)?
- [ ] Текст разложен по 5 битам (HOOK/SETUP/CONFLICT/TURN/INSIGHT)?
- [ ] Таймкоды расставлены по beat sheet?
- [ ] CU/MCU = 60-70% шотов?
- [ ] WS максимум 1-2?
- [ ] Нет двух одинаковых шотсайзов подряд?
- [ ] Каждый переход 2-3 сек, со звуковым мостом?
- [ ] Есть паузы (не забит голосом от начала до конца)?
- [ ] Ничего важного в нижних 20% кадра (UI зона)?

### Визуал и ритм
- [ ] Визуал-тест: закрой ГОЛОС, читай только ВИЗУАЛ — история считывается?
- [ ] Ритм-тест: нет трёх кадров подряд с одинаковым ритмом?
- [ ] Эмоция-тест: каждый кадр — другая эмоция зрителя?
- [ ] Визуальная метафора (если есть) — видна на картинке без текста?

### Арт-дирекция (refs/ART_DIRECTION.md)
- [ ] Style prefix вставлен в каждый MIDJOURNEY-промпт?
- [ ] Style suffix вставлен (свет + атмосфера)?
- [ ] --oref указан для каждого персонажа (не старый --cref)?
- [ ] Нет запрещённых слов (photorealistic, anime, dark, neon, hyperdetailed)?
- [ ] MOOD заполнен для каждого мульт-шота?
- [ ] Цветовая температура соответствует эмоции кадра?

### Голоса и аудио
- [ ] Каждая реплика размечена (эмоция + темп + акценты)?
- [ ] Telegram-реплики помечены (telegram)-тегом?
- [ ] Есть паузы после юмора (||пауза||)?
- [ ] Аудио-карта серии заполнена (5 бит × голос/музыка/SFX)?
- [ ] Музыка — один bed на серию?
- [ ] Есть СТОП (тишина) перед ключевым моментом?
- [ ] SFX: swoosh на переходах, Telegram-звук, макс 1 комедийный стингер?

### Съёмка Юрия
- [ ] Карточка шота заполнена для каждого реального шота (текст, выражение, экран, поза)?
- [ ] Continuity: шот 1 и шот 6 — один сетап, разный экран?
- [ ] Свет: окно сбоку, потолочный выключен?
- [ ] Кадр: глаза на верхней трети, ноутбук виден?
- [ ] Указано количество дублей (5-7)?

### Сквозные линии
