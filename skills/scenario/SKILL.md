# /scenario — Сценарист видеосериала

## Назначение
Получает историю (из MY_STORIES.md или свободный текст от Юрия) → пишет раскадровку для видео: визуал каждого кадра + текст озвучки + Midjourney-промпты. Как режиссёр-сценарист, не как копирайтер.

## Триггер
- Ручной: `bash skills/scenario/run.sh "название или текст истории"`
- Из Claude Code: Юрий говорит какую историю снимать

## Параметры
| Параметр | Тип | Обязательный | По умолчанию |
|----------|-----|-------------|-------------|
| story | string | да | - |
| --lang | string | нет | "ru" (варианты: ru, en, both) |
| --duration | string | нет | "short" (short=2мин, long=3мин) |

## Входные данные
| Источник | Зачем |
|----------|-------|
| Аргумент (story) | Название истории из MY_STORIES.md ИЛИ свободный текст |
| `~/Documents/WORK/AI_OFFICE/refs/MY_STORIES.md` | Банк реальных историй — факты, диалоги, уроки |
| `SERIES_BIBLE.md` | Персонажи, тон, формат, визуальный стиль, сквозные линии |
| `assets/characters/PROMPTS.md` | URL мастер-образов для --cref/--sref |

## КРИТИЧЕСКИЕ ПРАВИЛА

1. **Юрий выбирает историю.** Скилл НЕ выбирает сам. На входе — конкретная история или тема.
2. **Основа — реальная, подача — творческая.** Ситуация, цифры, цитаты Юрия — из MY_STORIES.md. Реплики персонажей-агентов, визуал, юмор, драматургия — работа сценариста. Это кино по реальным событиям, не документальная хроника.
3. **Раскадровка, не текст.** Выход = визуальный сценарий с покадровым описанием. Не эссе, не пост.
4. **Один инсайт на серию.** Если история содержит два инсайта — предложить разбить на две серии.
5. **Юмор обязателен.** Без юмора — переписать.
6. **Персонажи живые.** У каждого агента — характер, реплики, реакции (по SERIES_BIBLE.md). Скаут может возразить, QA может закатить глаза, Клодище может пробубнить оправдание. Это делает сериал сериалом.
7. **Сквозные линии.** В каждой серии — хотя бы один момент из сквозных линий (SERIES_BIBLE.md): химия Юрий-Today, конфликт скаутов, running gag. Максимум 1-2 момента. Если сквозная линия перетягивает инсайт — убрать.
8. **Show, don't tell.** Визуал рассказывает историю БЕЗ голоса. 80% зрителей смотрят без звука. Тест: закрой текст ГОЛОС, читай только ВИЗУАЛ по кадрам — если история не считывается, переписать визуал.

## Как скилл пишет сценарий (инструкция для промпта)

Ты — режиссёр-сценарист мини-сериала. Не копирайтер, не блогер.

**Принципы:**
- Думай кадрами, не абзацами. Каждый кадр = одна мысль + одна картинка.
- Зацепка — с первой секунды. Не "Привет, сегодня расскажу..." — а сразу в ситуацию.
- Диалоги — дословно из MY_STORIES.md. Они живее любого придуманного.
- Поворот — момент когда зритель не ожидал. "Я спросил — ты проверил? — Нет."
- Инсайт — не мораль, а наблюдение. "Ваши сотрудники делают то же самое. Только не признаются."
- Финал — вопрос или крючок. Не точка, а многоточие.

**Эмоциональный контраст:**
- Каждый кадр — своя эмоция зрителя. Два кадра подряд с одной эмоцией = провал.
- Кривая: узнавание → удивление → смех → задумался → "хочу ещё".
- В 2 минутах нет времени на ровное настроение — каждые 10-15 секунд эмоция меняется.

**Ритм (пульс серии):**
- Серия = пульс: быстро-быстро-ПАУЗА-быстро-УДАР-медленно.
- Правило: если три кадра подряд — одинаковый ритм — сломай один из них.
- Зацепка = быстро. Контекст = средне. Поворот = быстро. Инсайт = медленно. Финал = средне.

**Тон (из SERIES_BIBLE.md):**
- Как рассказываешь другу за кофе
- Самоирония обязательна
- Крепкое словцо допустимо — редко и метко
- Метафоры из физического мира

**Визуал (для Midjourney-промптов):**
- Стилизованные иллюстрации, тёплые тона
- Персонажи по описаниям из SERIES_BIBLE.md (цвета, силуэты!)
- Эмоции читаются с картинки (не нужен текст чтобы понять что происходит)
- Каждый кадр самодостаточен — можно показать без контекста
- Тест без звука: пролистай все ВИЗУАЛ подряд. История понятна? Если нет — переписать.

**Сквозные линии (из SERIES_BIBLE.md):**
- Вплети 1-2 момента из сквозных линий: химию Юрий-Today, конфликт скаутов, running gag.
- Естественно, не натужно. Если в эту серию не ложится — не заставляй.
- Today может появиться в одном кадре — и одной фразой задать тон всей серии.

## Верификация
- **Самопроверка:** Есть ли зацепка в первые 5 секунд? Есть ли поворот? Один ли инсайт? Есть ли юмор?
- **Факт-чек:** Все диалоги и факты — из MY_STORIES.md? Ничего не выдумано?
- **Хронометраж:** Прочитать текст озвучки вслух — укладывается в 2-3 минуты? (~300-450 слов для русского, ~250-400 для английского)
- **Визуал-тест:** Закрой ГОЛОС, читай только ВИЗУАЛ. История считывается? Если нет — переписать.
- **Ритм-тест:** Есть ли три кадра подряд с одинаковым ритмом? Если да — сломать один.
- **Эмоция-тест:** Каждый кадр — другая эмоция зрителя? Два одинаковых подряд — переписать.
