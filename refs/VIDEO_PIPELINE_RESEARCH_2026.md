## 3. GitHub-репозитории: рабочие пайплайны

### 3.1 ai-video-generation-workflow (222 звёзд)
- **URL:** https://github.com/haolinwang819-boop/ai-video-generation-workflow
- **Стек:** Node.js + TypeScript, FFmpeg, Gemini API, ElevenLabs, edge-tts
- **Пайплайн:** Скрипт → Слайды → TTS → Субтитры → Таймлайн → Рендер MP4
- **Релевантность:** Высокая — похож на наш assemble.py, но модульнее
- **Полезно:** Checkpoint recovery, reruns отдельных этапов, настраиваемый TTS

### 3.2 VideoGraphAI (50 звёзд)
- **URL:** https://github.com/mikeoller82/VideoGraphAI
- **Стек:** Groq/OpenAI + TogetherAI (FLUX) + F5-TTS + Tavily Search + Streamlit + FFmpeg
- **Пайплайн:** Ресерч → Скрипт → Сториборд → Картинки → Голос → Субтитры → Видео
- **Релевантность:** Средняя — ориентирован на YouTube Shorts, но архитектура полезна

### 3.3 Timeline Studio (148 звёзд)
- **URL:** https://github.com/chatman-media/timeline-studio
- **Стек:** Next.js 15 + Rust (Tauri v2) + FFmpeg, Claude/OpenAI/DeepSeek
- **Пайплайн:** AI-видеоредактор с 100+ инструментами, экспорт на TikTok/YouTube/Telegram
- **Релевантность:** Средняя — больше редактор, чем генератор

### 3.4 MaxVideoAI (22 звёзд)
- **URL:** https://github.com/camgraphe/MaxVideoAi
- **Стек:** Next.js + fal.ai (Sora, Veo, Kling, Seedance, Wan)
- **Назначение:** Сравнение видеогенераторов side-by-side, единый интерфейс
- **Релевантность:** Полезно для тестирования моделей

### 3.5 video-notation-schema (9 звёзд)
- **URL:** https://github.com/context-notation/video-notation-schema
- **Формат:** JSON-схема для структурированных промптов к видео-AI
- **Релевантность:** Интересная идея для стандартизации раскадровки

### 3.6 Open-Higgsfield-AI (2.6k звёзд)
- **URL:** https://github.com/Anil-matcha/Open-Higgsfield-AI
- **Стек:** Next.js + Muapi.ai (200+ моделей: Kling, Sora, Veo, Runway, Seedance...)
- **Назначение:** Единый интерфейс для генерации изображений и видео через API
- **Релевантность:** Высокая — можно генерить видео из картинок Midjourney

## 4. ТОП-3 пайплайна: сравнение

### Пайплайн A: "Slideshow+" (текущий, улучшенный)

**Стек:** Midjourney (картинки) → ElevenLabs (голос) → Suno (музыка) → assemble.py (сборка)

**Улучшения:**
- Добавить Ken Burns эффект (зум/панорамирование) — уже может быть в assemble.py
- Добавить субтитры (Whisper для таймингов)
- Добавить переходы между сценами

| | |
|---|---|
| **Стоимость** | ~$30-50/мес (Midjourney $10 + ElevenLabs $5-22 + Suno $10) |
| **Качество** | Среднее — slideshow, но с хорошими картинками и голосом |
| **Скорость** | 1-2 часа на серию |
| **Контроль** | Максимальный — каждый кадр контролируешь |
| **Плюсы** | Дёшево, предсказуемо, полный контроль, работает сейчас |
| **Минусы** | Нет движения в кадре, выглядит как презентация |
| **Для кого** | Минимальный бюджет, быстрый старт, контентная стратегия |

### Пайплайн B: "Hybrid Motion" (картинки → анимация через AI)

**Стек:** Midjourney/DALL-E (картинки) → Kling/Runway (img2vid — оживление) → ElevenLabs (голос) → FFmpeg/assemble.py (сборка)

**Как работает:**
1. Генерим статичные картинки (Midjourney) — контроль стиля
2. Каждую картинку "оживляем" через img2vid (3-5 секунд движения)
3. Собираем с голосом и музыкой

| | |
|---|---|
| **Стоимость** | ~$60-120/мес (Midjourney $10 + Kling/Runway $25-76 + ElevenLabs $5-22 + Suno $10) |
| **Качество** | Высокое — движение в кадре, кинематографичность |
| **Скорость** | 3-5 часов на серию (генерация видео медленная) |
| **Контроль** | Высокий — основа из картинок, анимация добавляет жизнь |
| **Плюсы** | Живое видео, консистентный стиль (из одних картинок), впечатляет зрителя |
| **Минусы** | Дороже, некоторые кадры могут "поплыть", требует перегенерации |
| **Для кого** | Баланс качества и цены, профессиональный вид |

**Лучшие комбинации img2vid:**
- **Kling V3-Omni** — лучшее качество/цена через API, стабильная анимация
- **Runway Gen-4.5** — кинематографичность, но дороже
- **Hailuo 2.3** — хорошая плавность, доступная цена
- **Seedance 2.0** — новый, но впечатляющие результаты

### Пайплайн C: "Full AI Video" (полная генерация видео)

**Стек:** Claude (скрипт+промпты) → Veo 3/Sora/Kling (txt2vid — полные сцены) → ElevenLabs (голос) → FFmpeg (сборка)

**Как работает:**
1. Пишем детальные промпты для каждой сцены
2. Генерим видеоклипы напрямую из текста (5-10с каждый)
3. Собираем с голосом и музыкой

| | |
|---|---|
| **Стоимость** | ~$50-150/мес (Veo/Sora через подписки + ElevenLabs + Suno) |
| **Качество** | Потенциально высшее — полноценное видео |
| **Скорость** | 4-8 часов (много перегенераций для нужного результата) |
| **Контроль** | Низкий — трудно получить точно то что нужно |
| **Плюсы** | Самое впечатляющее видео, полное движение, кинематограф |
| **Минусы** | Нет консистентности персонажей, много мусорных генераций, дорого на итерации |
| **Для кого** | Экспериментаторы, когда консистентность стиля не критична |

## 5. Рекомендация для AI_SHOW

### Оптимальный путь: Пайплайн B "Hybrid Motion"

**Почему:**
1. **Консистентность** — Midjourney даёт стабильный стиль персонажей (через CREF и стайл-промпты)
2. **Wow-эффект** — Оживлённые картинки выглядят на порядок лучше slideshow
3. **Контроль** — Ты контролируешь визуал на этапе картинки, движение добавляется сверху
4. **Цена** — $60-80/мес вполне подъёмна, особенно через API Kling
5. **Минимум изменений** — Основной пайплайн остаётся, добавляется один шаг

### План перехода:

**Фаза 1 (сейчас):** Продолжать Пайплайн A — выпускать серии
**Фаза 2 (тест):** Взять 2-3 готовые картинки S01E01 → прогнать через Kling img2vid → оценить качество
**Фаза 3 (интеграция):** Добавить в assemble.py поддержку видеоклипов вместо статичных картинок
**Фаза 4 (продакшн):** S01E02+ собирать через Hybrid Motion пайплайн

### Конкретные шаги для теста:

1. Зарегистрироваться на klingai.com — бесплатные кредиты на старте
2. Загрузить 3 картинки из `assets/scenes/S01E01/`
3. Сгенерить 5-секундные видео (img2vid)
4. Оценить: консистентность стиля, плавность, артефакты
5. Сравнить с Hailuo (hailuoai.video) — тоже бесплатные кредиты на старте

### Что нужно доработать в assemble.py:

- Поддержка входных видеоклипов (MP4) вместо только PNG/JPG
- Микширование: видеоклип + голос + фоновая музыка
- Fallback: если для кадра нет видео — использовать статичную картинку с Ken Burns

## 7. Полезные ресурсы

### Open-source пайплайны:
- `ai-video-generation-workflow` — модульный пайплайн с checkpoint recovery
- `Open-Higgsfield-AI` — единый интерфейс для 200+ моделей генерации

- **ComfyUI** — нодовый интерфейс для Seedance, Kling и др. (локально)

- Отдельная категория: AI-аватары для говорящей головы
- 125+ аватаров, 1200+ голосов, 140+ языков
- НЕ подходит для стилизованной анимации AI_SHOW, но полезно знать
