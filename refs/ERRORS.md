## #001 — Аудит: ложное срабатывание (E6 — IMG2VID)
**Дата:** 28 марта 2026
**Что случилось:** При аудите S01E02 отметил "нет IMG2VID для шота 4" как проблему. IMG2VID был на месте (строки 113-116).
**Причина:** Проверял по памяти вместо перечитывания файла.
**Решение:** Правило "Сначала Read" работает и для аудита. Каждый пункт чек-листа — grep или Read, не по памяти.
**Паттерн:** Аудит по памяти → ложные срабатывания.

## #002 — Bible vs практика: переходы Seedance vs CapCut
**Дата:** 28 марта 2026
**Что случилось:** SERIES_BIBLE требовал Seedance-клипы для переходов. S01E01 (утверждённый пилот) уже использовал CapCut-переходы. Расхождение документации и практики.
**Причина:** Bible написан как идеал до продакшена. Пилот установил практический стандарт.
**Решение:** Обновил SERIES_BIBLE строка 32 — разрешил оба варианта.
**Паттерн:** Документация отстаёт от практики. После каждого пилота — сверять Bible с реальностью.

## #003 — Промпты: неполный style suffix
**Дата:** 28 марта 2026
**Что случилось:** Оверлейные промпты (шоты 1Б, 4Б) не содержали полный style suffix из ART_DIRECTION.md.
**Причина:** Оверлеи воспринимались как "вспомогательные" картинки, не основные кадры.
**Решение:** Style prefix + suffix — в КАЖДЫЙ промпт без исключений. Даже оверлеи.
**Паттерн:** "Второстепенный" элемент = тот же стандарт качества.

## #004 — Промпты: пропущены --oref для персонажей
**Дата:** 28 марта 2026
**Что случилось:** В групповых сценах (3+ персонажа) --oref не был указан.
**Причина:** Непонятно как указывать --oref для нескольких персонажей в одном промпте.
**Решение:** `--oref [file1.png URL] [file2.png URL] [file3.png URL] --ow 300` — Midjourney V7 принимает несколько референсов.
**Паттерн:** Групповые сцены → обязательно перечислить всех видимых персонажей в --oref.
