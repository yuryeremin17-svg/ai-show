## 1. ТИПЫ ШОТОВ И КОГДА ИСПОЛЬЗОВАТЬ

### Таблица шотов

| Тип шота | Что в кадре | Когда использовать | Эмоция зрителя | Пример AI Office |
|----------|-------------|-------------------|----------------|-----------------|
| **ECU** (Extreme Close-Up) | Глаза, рот, деталь (кнопка, экран) | Ключевой момент, шок, открытие | Интимность, напряжение | Глаза Клодища — распахиваются при прочтении сообщения |
| **CU** (Close-Up) | Лицо от макушки до подбородка | Эмоция, реакция, инсайт | Эмпатия, связь с персонажем | Юрий смотрит в камеру — "Дело не в сотруднике" |
| **MCU** (Medium Close-Up) | Голова + плечи | Диалог, реплика | Разговор, доверие | Клодище поправляет очки — "Перестраховался" |
| **MS** (Medium Shot) | От головы до пояса | Основное действие, жесты | Нейтральное вовлечение | Клодище скрещивает руки, читает сообщение |
| **MWS** (Medium Wide) | Персонаж с окружением | Контекст + действие | Понимание ситуации | Юрий за столом с ноутбуком, Дубай за окном |
| **WS** (Wide Shot) | Полная фигура + окружение | Установка места, переход | Масштаб, "где мы?" | Мультяшный офис целиком — агенты за столами |
| **EWS** (Extreme Wide) | Панорама, здание | Только открывающий/закрывающий кадр | Эпичность, итог | Дубай снаружи (НЕ для 60 сек — слишком долго) |

### Правила выбора шота для 60-секундного видео

```
ПРАВИЛО 1: CU и MCU — 60-70% шотов.
  В вертикальном 9:16 лицо — главный актив.
  Мелкие детали теряются. Крупный план работает.

ПРАВИЛО 2: WS — максимум 1-2 раза за серию.
  Только для установки мира (первый кадр мультика)
  или финала. Не тратить время на панорамы.

ПРАВИЛО 3: Каждая смена шота = смена информации.
  Если CU не добавляет новую эмоцию — не нужен.
  Если WS не даёт новый контекст — убрать.

ПРАВИЛО 4: Чередование масштаба.
  Никогда два CU подряд. Никогда два WS подряд.
  Паттерн: MS → CU → MS → MCU → WS → CU.

ПРАВИЛО 5: Вертикальный кадр = вертикальная композиция.
  Персонаж занимает 60-70% высоты кадра.
  Пустое пространство — сверху (воздух) или снизу (субтитры).
  Горизонтальные элементы (стол, окно) — фоном.
```

### Специфика 9:16 (вертикальное видео)

```
┌──────────────┐
│   SAFE ZONE  │ ← верхние 10% свободны от UI
│              │
│  ┌────────┐  │
│  │ОСНОВНОЙ│  │ ← центральные 60% — ключевой контент
│  │КОНТЕНТ │  │
│  │  ЛИЦО  │  │
│  └────────┘  │
│              │
│ ⚠ DANGER   │ ← нижние 20% закрыты UI (подписи, кнопки)
│   ZONE      │ ← НЕ размещать важный текст/детали
│  [♥ 💬 ➤]  │ ← правый край закрыт кнопками соцсетей
└──────────────┘

ПРАВИЛА:
- Лицо персонажа — в верхней трети кадра (правило третей)
- Субтитры — в центральной зоне, НЕ внизу (закроет UI)
- Жесты рук — видны только если MCU или шире
- Текст на экранах (монитор Клодища) — крупно, читаемо
```

## 2. КАМЕРА: УГЛЫ И ИХ ЭМОЦИОНАЛЬНОЕ ЗНАЧЕНИЕ

### Таблица углов камеры

| Угол | Описание | Эмоция | Когда в AI Office |
|------|----------|--------|-------------------|
| **Eye Level** | Камера на уровне глаз | Нейтрально, равенство, доверие | Юрий на камеру (зацепка, финал). Основной угол. 80% шотов |
| **Slight Low** (5-15°) | Чуть снизу | Уверенность, авторитет | Клодище когда "включается" — принял задачу, глаза горят |
| **Low Angle** (15-45°) | Снизу вверх | Сила, доминирование, угроза | РЕДКО. Можно для "босс-момента" Юрия |
| **Slight High** (5-15°) | Чуть сверху | Уязвимость, открытость | Клодище когда "перестраховался" — пойман, виноват |
| **High Angle** (15-45°) | Сверху вниз | Слабость, подавленность | Агент облажался, провал, комедийный момент |
| **Bird's Eye** (90°) | Строго сверху | Обзор, схема, контроль | Рабочий стол с документами (insert shot) |
| **Dutch Angle** (наклон) | Кадр наклонён 10-25° | Тревога, что-то не так, безумие | Когда всё идёт не по плану. Лёгкий наклон = юмор |
| **OTS** (Over-the-Shoulder) | Через плечо | Диалог, противостояние | Клодище смотрит на монитор с сообщением Юрия |

### Правила для AI Office

```
ПРАВИЛО 1: Eye Level — по умолчанию.
  Юрий всегда на eye level (он говорит как равный с аудиторией).
  Агенты — eye level для нейтральных моментов.

ПРАВИЛО 2: Угол = эмоциональный сдвиг.
  Смена угла камеры = что-то изменилось.
  Не менять угол ради разнообразия. Менять ради значения.

ПРАВИЛО 3: Slight High/Low — основной инструмент.
  Резкие углы (45°+) — для комедии или драмы.
  В edutainment 60 сек — лёгкие сдвиги (5-15°).

ПРАВИЛО 4: Dutch Angle — только для юмора.
  В нашем жанре Dutch = "ну и каша".
  Максимум 1 раз за серию. Лёгкий наклон (10-15°).

ПРАВИЛО 5: Движение камеры в Seedance.
  Push-in (наезд) = усиление момента, фокусировка.
  Pull-back (отъезд) = раскрытие контекста, удивление.
  Static + slight sway = нейтральный, спокойный.
  Orbit (облёт) = энергия, трансформация.
```

## 3. ТАЙМИНГ: ДАННЫЕ И ПРАВИЛА

### Оптимальная длительность видео по платформам (данные 2025-2026)

| Платформа | Развлечение | Образование | Сторителлинг |
|-----------|-------------|-------------|--------------|
| **TikTok** | 18-31 сек | 42-54 сек | 60-90 сек |
| **Instagram Reels** | 11-17 сек | 38-51 сек | 60-90 сек |
| **YouTube Shorts** | 28-33 сек | 41-57 сек | До 60 сек |
| **LinkedIn** | 40-46 сек (short) | 3-4 мин (long) | — |

**AI Office = edutainment-сторителлинг. Оптимум: 42-60 секунд.**

### Average Shot Length (ASL) — данные

| Контекст | ASL (сек на шот) | Источник |
|----------|-------------------|----------|
| Коммерческая реклама | 2-3 сек | Industry standard 2025 |
| Музыкальный клип (припев) | 3.5 сек | Top charts 2024 |
| Музыкальный клип (куплет) | 5-6 сек | Top charts 2024 |
| YouTube Shorts (top 100) | 2.5 сек | 2024 data analysis |
| Короткометражная анимация | 4-6 сек | Pixar/animation standard |
| Edutainment (Kurzgesagt) | 5-8 сек | Analysis of videos |
| Диалоговая сцена | 6-10 сек | Film industry norm |
| **AI Office (рекомендация)** | **5-10 сек** | **Наш формат** |

### Формула для AI Office: 60 секунд = 6-8 шотов

```
60 сек / ASL 8 сек = 7.5 шотов → 6-8 шотов ✓ (совпадает с SERIES_BIBLE)

Распределение:
- 2 шота реального видео (начало + конец): ~20 сек
- 2 перехода между мирами: ~6 сек
- 3-4 мульт-шота: ~34 сек
```

### Правило "Heartbeat Edit" для соцсетей

```
Каждые 2-3 секунды — визуальное изменение ВНУТРИ шота.
Это НЕ смена шота. Это:
  - Смена выражения лица
  - Появление текста/пузыря чата
  - Движение камеры (push-in, pull-back)
  - Анимация элемента (иконка, уведомление)
  - Смена фокуса

Если 5 секунд ничего не меняется — зритель уходит.
В Seedance-промптах: всегда описывать движение.
В реальном видео Юрия: жестикуляция, взгляд на экран, поворот.
```

## 4. BEAT SHEET ДЛЯ 60-СЕКУНДНОГО ВИДЕО

### Структура: 5 бит за 60 секунд

| Бит | Время | Длительность | Что происходит | Темп | ASL |
|-----|-------|-------------|----------------|------|-----|
| **1. HOOK** | 0:00-0:03 | 3 сек | Первая фраза. Контраст, шок, вопрос. Решение "остаться/уйти" | БЫСТРО | 3 сек |
| **2. SETUP** | 0:03-0:13 | 10 сек | Контекст ситуации. Кто, где, что. Переход в мир агентов | СРЕДНЕ | 5-7 сек |
| **3. CONFLICT** | 0:13-0:35 | 22 сек | Проблема, реакция агентов, нарастание. Основное действие | НАРАСТАЕТ | 7-10 сек |
| **4. TURN** | 0:35-0:50 | 15 сек | Поворот/решение. Кульминация + юмор. Переход обратно | БЫСТРО→ПАУЗА | 5-8 сек |
| **5. INSIGHT** | 0:50-1:00 | 10 сек | Инсайт Юрия. Наблюдение, не мораль. Крючок | МЕДЛЕННО | 10 сек |

### Beat Sheet — наложение на структуру "Два мира"

```
SEC  BEAT        WORLD     SHOT TYPE    CAMERA         EMOTION
───  ──────────  ────────  ──────────   ─────────────  ────────────────
0    HOOK        ЮРИЙ      CU/MCU       Eye Level      Узнавание
3    SETUP       ЮРИЙ      MWS          Eye Level      Любопытство
10   TRANSITION  →МУЛЬТ    WS→CU        Push-in/Dive   Погружение
13   CONFLICT-1  АГЕНТЫ    MS/MCU       Eye Level      Удивление
22   CONFLICT-2  АГЕНТЫ    CU           Slight High    Напряжение
30   TURN        АГЕНТЫ    MCU→CU       Slight Low     Смех / "ага!"
40   RESOLUTION  АГЕНТЫ    MS           Eye Level      Облегчение
47   TRANSITION  →РЕАЛ     CU→MWS       Pull-back      Возвращение
50   INSIGHT     ЮРИЙ      CU           Eye Level      Задумался
57   HOOK-OUT    ЮРИЙ      CU           Eye Level      "Хочу ещё"
```

```

  │         ╭──╮
  │        ╱    ╲    ╭─╮
  │   ╭──╮╱      ╲  ╱   ╲
```

## 5. ТАЙМИНГ ХУКА (0-3 секунды) — КРИТИЧНО

### Данные

- **65%** людей, посмотревших первые 3 сек, досмотрят до 10 сек
- **45%** досмотрят до 30 сек
- 3-second hold — ключевая метрика алгоритма Instagram
- Хук должен быть произнесён за **2-2.5 секунды** (быстрый темп)

### Структура хука для AI Office

```
ВИЗУАЛ (0.0 сек): Юрий уже в кадре. Никаких заставок.
ГОЛОС  (0.0 сек): Первое слово = сразу в ситуацию.
ТЕКСТ  (0.5 сек): Субтитр с ключевым словом появляется.
СДВИГ  (2.0 сек): Что-то меняется — выражение лица, жест, взгляд на экран.
ОБЕЩАНИЕ (3.0 сек): Зритель понимает "о чём будет" и хочет узнать "чем кончится".

АНТИПАТТЕРН:
  ✗ "Привет, сегодня расскажу..."
  ✗ Заставка с логотипом
  ✗ Тишина первые 0.5 сек
  ✗ Медленный зум-ин на лицо
```

## 6. ПРАВИЛА КОМПОЗИЦИИ ДЛЯ ВЕРТИКАЛЬНОГО КАДРА

### Rule of Thirds (9:16)

```
┌────────┬────────┬────────┐
│        │  глаза │        │  ← верхняя линия третей:
│        │ [здесь]│        │    глаза персонажа
├────────┼────────┼────────┤
│        │        │        │  ← средняя линия:
│        │ [руки] │        │    жесты, объекты в руках
├────────┼────────┼────────┤
│        │        │        │  ← нижняя линия:
│        │[стол/  │        │    контекст, поверхности
│        │ экран] │        │
└────────┴────────┴────────┘

ПРАВИЛО: глаза на верхней линии третей = естественное внимание.
В 9:16 это работает сильнее чем в 16:9.
```

### Leading Lines в вертикальном кадре

```
РАБОТАЕТ:
- Вертикальные линии (окна, стены, мониторы) — усиливают формат
- Диагонали (рука указывает, взгляд направлен) — энергия
- Экран ноутбука как рамка-в-рамке — фокус на контент

НЕ РАБОТАЕТ:
- Горизонтальные панорамы — обрезаются в 9:16
- Симметрия — скучно в вертикальном формате
- Мелкие детали по краям — закроют кнопки UI
```

### Фокальная точка (правило одного взгляда)

```
ПРАВИЛО: зритель должен за 0.5 сек понять куда смотреть.

  1 персонаж = он в центре, контрастный к фону
  2 персонажа = один выше другого (не рядом — вертикаль!)
  Текст на экране = крупно, контрастно, в safe zone

В промптах Midjourney:
  "focal point on [главный элемент]"
  "blurred background, sharp foreground character"
  "centered composition, vertical framing"
```

## 7. СПЕЦИФИКА MIXED MEDIA (два мира)

### Переходы реал-мульт: правила

| Элемент | Правило | Почему |
|---------|---------|--------|
| **Длительность** | 2-3 сек на переход | Дольше = потеря ритма. Короче = непонятно |
| **Направление** | Реал→Мульт: dive in (наезд в экран). Мульт→Реал: zoom out (отъезд из экрана) | Консистентный паттерн, зритель привыкает |
| **Якорь** | Экран ноутбука = портал между мирами | Один объект связывает два мира |
| **Звук** | Звуковой мост: звук начинается ДО визуального перехода | Подготавливает мозг к смене |
| **Цвет** | Реал = натуральные цвета. Мульт = тёплые, насыщенные | Контраст подчёркивает разницу миров |
| **Частота** | 2 перехода за 60 сек (не больше) | Каждый переход = 2-3 сек потери. Больше 2 = слишком рвано |

### Storyboard-формат для mixed media шота

```markdown
## ШОТ [N]T — ПЕРЕХОД: РЕАЛ → МУЛЬТ [0:XX-0:XX]

МОСТ:        [что связывает два мира: экран, звук, объект]
ВИЗУАЛ-РЕАЛ: [последний кадр реального мира]
ДВИЖЕНИЕ:    [dive-in / zoom-out / morph / match cut]
ВИЗУАЛ-МУЛЬТ:[первый кадр мультяшного мира]
ЗВУК:        [звуковой мост: что начинается ДО перехода]
SEEDANCE:    [промпт для Seedance — генерирует переход как клип]

Длительность: 2-3 сек
```

### Match Cut между мирами (продвинутый приём)

```
Приём: последний кадр реального мира = первый кадр мультяшного мира,
       но в другом стиле.

Примеры:
  Реал: Юрий нажимает Enter на клавиатуре
  Мульт: Палец Клодища нажимает кнопку на мониторе

  Реал: Кофейная чашка Юрия
  Мульт: Кофейная чашка на столе мультяшного офиса

  Реал: Юрий смотрит на экран, лицо крупно
  Мульт: Клодище смотрит на экран, лицо крупно (тот же угол)
```

## 9. ТАЙМИНГ ОЗВУЧКИ И ПОДСЧЁТ СЛОВ

### Скорость речи (русский язык)

| Темп | Слов/мин | Символов/мин | Когда |
|------|----------|-------------|-------|
| Медленный | 100-110 | ~600 | Инсайт, финал, пауза для осмысления |
| Нормальный | 120-140 | ~800 | Основной нарратив, контекст |
| Быстрый | 150-170 | ~1000 | Хук, нарастание, энергия |

### Формула подсчёта для 60 секунд

```
60 сек видео ≠ 60 сек речи.

Реальное распределение:
  Речь:     ~45 сек (80-130 слов русский)
  Паузы:    ~8 сек (после поворота, перед инсайтом, для смеха)
  Тишина:   ~4 сек (переходы между мирами)
  SFX only: ~3 сек (уведомления, звуки офиса)

АНТИПАТТЕРН: заполнить все 60 сек голосом.
  Паузы = пространство для эмоции и юмора.
```

### Тайминг по битам

| Бит | Секунд | Слов (рус) | Темп речи |
|-----|--------|-----------|-----------|
| HOOK (0-3) | 3 сек | 8-12 слов | Быстрый |
| SETUP (3-13) | 10 сек | 20-25 слов | Нормальный |
| CONFLICT (13-35) | 22 сек | 35-45 слов | Нарастающий |
| TURN (35-50) | 15 сек | 15-25 слов + пауза | Быстро→Пауза |
| INSIGHT (50-60) | 10 сек | 15-20 слов | Медленный |
| **ИТОГО** | **60 сек** | **93-127 слов** | **Переменный** |

## 10. ШПАРГАЛКА ДЛЯ РАСКАДРОВЩИКА (одна страница)

### Быстрый чек перед сдачей раскадровки

```
СТРУКТУРА:
  ✓ 6-8 шотов (включая 2 перехода)?
  ✓ ASL = 5-10 сек?
  ✓ Хук за 3 секунды?
  ✓ Поворот в районе 0:30-0:40?
  ✓ Инсайт в последние 10 сек?

SHOT MIX:
  ✓ CU/MCU = 60-70% шотов?
  ✓ WS = максимум 1-2 шота?
  ✓ Нет двух одинаковых shot sizes подряд?
  ✓ Углы камеры обоснованы эмоцией?

ТАЙМИНГ:
  ✓ 80-130 слов русский текст?
  ✓ Есть паузы (не забит голосом)?
  ✓ Каждые 2-3 сек — визуальное изменение?
  ✓ Ритм чередуется (быстро/средне/пауза)?

ВЕРТИКАЛЬ:
  ✓ Глаза на верхней трети кадра?
  ✓ Ничего важного в нижних 20%?
  ✓ Фокальная точка за 0.5 сек?
  ✓ Субтитры в safe zone?

ДВА МИРА:
  ✓ 2 перехода (не больше)?
  ✓ Каждый переход 2-3 сек?
  ✓ Звуковой мост перед переходом?
  ✓ Якорь (экран ноутбука) в обоих переходах?
  ✓ Цветовой контраст реал/мульт?

ЭМОЦИИ:
  ✓ Каждый шот — новая эмоция?
  ✓ Кривая: узнавание→удивление→смех→задумался→хочу ещё?
  ✓ Есть пауза для смеха после юмора?
```

## 11. ПРИМЕНЕНИЕ К ТЕКУЩЕМУ ФОРМАТУ

### Что добавить в подроль 2 (Раскадровщик) скилла /scenario

Текущий формат раскадровки (S01E01_ru.md) содержит:
- ТИП, ВИЗУАЛ, ГОЛОС, MIDJOURNEY, IMG2VID

Рекомендуемые дополнения:
1. **ШОТСАЙЗ** — конкретный тип шота (CU, MS, WS)
2. **УГОЛ КАМЕРЫ** — эмоциональное обоснование
3. **ДВИЖЕНИЕ** — направление камеры в Seedance
4. **РИТМ** — быстро/средне/медленно
5. **HEARTBEAT** — что меняется внутри шота каждые 2-3 сек
6. **ЗВУК** — SFX, музыка, тишина (отдельно от голоса)
7. **СУБТИТРЫ** — ключевые слова для выделения

### Пример: S01E01 ШОТ 3 (текущий vs улучшенный)

**Текущий:**
```
## ШОТ 3 — МИР АГЕНТОВ: ПОВОРОТ [0:22-0:32]
ТИП: мультик (Midjourney → Seedance)
ВИЗУАЛ: Тот же офис. Звук Telegram-уведомления...
```

**Улучшенный:**
```
## ШОТ 3 — МИР АГЕНТОВ: ПОВОРОТ [0:22-0:32]
ТИП:          мультик (Midjourney → Seedance)
ШОТСАЙЗ:      MCU → CU (наезд в процессе шота)
УГОЛ КАМЕРЫ:  Eye Level → Slight Low (энергия включается)
ДВИЖЕНИЕ:     Push-in на лицо при прочтении сообщения
КОМПОЗИЦИЯ:   Клодище в центре, монитор с сообщением — ведущая линия слева

ВИЗУАЛ:       Тот же офис. Звук Telegram-уведомления. На мониторе —
              новое сообщение: "А как БЕЗ дизайнера? Дай 3 решения."
              Клодище распахивает глаза. Наклоняется к экрану.
ГОЛОС ЮРИЯ (Telegram): "Хорошо. А как сделать БЕЗ дизайнера? Дай три решения."
ЗВУК:         Telegram notification → тишина 0.5 сек → голос
СУБТИТРЫ:     выделить: "БЕЗ дизайнера" "три решения"

ЭМОЦИЯ:       Удивление → интерес (зритель и Клодище вместе)
РИТМ:         БЫСТРО (поворотный момент)
HEARTBEAT:    0 сек: звук уведомления | 2 сек: глаза распахиваются |
              4 сек: наклон к экрану | 6 сек: голос Юрия

MIDJOURNEY:   [промпт]
SEEDANCE:     [промпт]
```
