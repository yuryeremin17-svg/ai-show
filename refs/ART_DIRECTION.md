## 1. Style Bible (стилевая ДНК)

### Что это
Style Bible -- фиксированный набор слов, который вставляется в КАЖДЫЙ промпт без изменений.
Это "ДНК стиля". Один раз определил -- больше не трогаешь.

### Style Bible для AI Office

**STYLE PREFIX (вставлять в начало каждого промпта):**
```
stylized warm illustration, soft 3D rendering, Pixar-style character design,
```

**STYLE SUFFIX (вставлять в конец каждого промпта, перед параметрами):**
```
warm golden lighting, soft ambient shadows, cozy professional atmosphere, matte finish, subtle depth of field --ar 9:16
```

**Полный шаблон промпта:**
```
[STYLE PREFIX] + [описание сцены/персонажа] + [STYLE SUFFIX] + [--sref/--oref параметры]
```

**Пример:**
```
stylized warm illustration, soft 3D rendering, Pixar-style character design, tall thin man in black turtleneck with round glasses standing at whiteboard explaining diagram to colleagues in modern office with Dubai skyline, warm golden lighting, soft ambient shadows, cozy professional atmosphere, matte finish, subtle depth of field --ar 9:16 --oref [claude.png URL] --ow 400
```

### Принцип: меньше слов = меньше дрифта
Больше стилевых слов -- больше "стилевого супа". Модель усредняет все токены,
и результат непредсказуем. 8-12 стилевых токенов -- потолок. Наш prefix+suffix
укладывается в это правило.

## 2. Цветовая палитра

### Теория (из практики Pixar)
Pixar использует "colour scripts" -- цветовые раскадровки, где каждая сцена
привязана к эмоции через цвет. Тёплые тона = комфорт, связь, позитив.
Холодные = одиночество, конфликт, неизвестность.

Для AI Office: основной тон ТЁПЛЫЙ (офис -- живой, дружелюбный).
Холодные акценты только в моментах конфликта или напряжения.

## 3. Словарь освещения

### Основные термины (что понимает AI)

**Для тёплого уютного стиля AI Office:**

| Термин | Эффект | Когда использовать |
|--------|--------|-------------------|
| `warm golden hour lighting` | Мягкий золотой свет, длинные тени | Основной стиль, 80% кадров |
| `soft ambient light` | Рассеянный мягкий свет без резких теней | Диалоги, крупные планы |
| `warm studio lighting` | Равномерный студийный свет, тёплый | Мастер-образы персонажей |
| `morning light` | Свежий утренний свет, чуть прохладнее | Сцены с Today (утренний брифинг) |
| `warm screen glow` | Свечение от экрана, тёплое | Ночные/вечерние сцены |
| `cozy interior lighting` | Лампы, мягкие тени, домашний уют | Камерные сцены |

**Для драматических моментов:**

| Термин | Эффект | Когда использовать |
|--------|--------|-------------------|
| `rim lighting` | Контурный свет, отделяет от фона | Момент инсайта, "ага!" |
| `volumetric lighting` | Видимые лучи света (через пыль/окно) | Эпические моменты |
| `dramatic side lighting` | Боковой свет, глубокие тени | Конфликт, напряжение |
| `backlit with warm glow` | Контровой свет, тёплый ореол | Герой принимает решение |
| `low key lighting` | Много теней, минимум света | Редко. Только для "ночных" flashback |

**Запрещённые типы освещения:**

| НЕ использовать | Почему |
|-----------------|--------|
| `harsh lighting` | Разрушает мягкость Pixar-стиля |
| `fluorescent lighting` | Холодный, офисный, некинематографичный |
| `flat lighting` | Убивает объём, делает "плоским" |
| `neon lighting` | Не наш стиль, уводит в киберпанк |
| `strobe lighting` | Агрессивный, неуместный |

### Температура света (для промптов)
- **Тёплый (наш основной):** warm, golden, amber, honey-toned, sunset
- **Нейтральный (редко):** natural daylight, soft white, balanced
- **Холодный (НЕ использовать):** cool blue, moonlight, sterile white

## 4. Список запрещённых элементов (Forbidden List)

### Зачем это нужно
AI-модели "дрифтуют" -- от промпта к промпту стиль уплывает.
Запрещённый список -- это якорь, который не даёт уплыть.

### Запрещённые стили и эстетики

```
НЕ использовать в промптах (ни в позитивной, ни в негативной форме):

СТИЛЬ:
- photorealistic / hyper-realistic / photograph
- anime / manga / cel-shaded
- low poly / voxel / pixel art
- gothic / dark fantasy / horror
- cyberpunk / synthwave / neon
- watercolor / oil painting / sketch
- flat design / vector / minimalist
- grunge / distressed / vintage filter

ПЕРСОНАЖИ:
- realistic human skin texture
- uncanny valley features
- exaggerated anime eyes
- chibi / super-deformed
- realistic wrinkles and pores
```

### Negative prompt (для Stable Diffusion / при необходимости)

```
photorealistic, photograph, anime, manga, dark, gothic, horror,
neon, cyberpunk, watercolor, sketch, pencil drawing, flat design,
vector art, low quality, blurry, grainy, jpeg artifacts, watermark,
text overlay, logo, extra limbs, deformed hands, bad anatomy,
disfigured face, asymmetric features, multiple heads
```

### Стоп-слова для сценариста
Если эти слова появляются в промпте -- перепроверить:

| Стоп-слово | Риск | Замена |
|------------|------|--------|
| `realistic` | Уводит в фотореализм | `stylized`, `illustrated` |
| `detailed` | Перегружает деталями | `clean`, `clear` |
| `complex` | Хаотичная композиция | `focused`, `simple composition` |
| `dark` | Уводит тональность | `moody`, `dramatic` (если нужно) |
| `bright neon` | Уводит в киберпанк | `warm golden`, `honey-toned` |
| `abstract` | Теряется читаемость | конкретное описание |
| `surreal` | Ломает "офисный" мир | `imaginative`, `playful` |

## 5. Метод Style Lock

### Три уровня фиксации стиля

**Уровень 1: Prompt DNA (текстовый)**
Фиксированный prefix + suffix в каждом промпте (см. раздел 1).
Работает везде. Минимальный уровень.

**Уровень 2: Visual Reference (визуальный)**
Использовать мастер-образы и палитру как референсы.

Для Midjourney V7:
```
--oref [master_image_URL] --ow 300-500    для персонажей
--sref [style_reference_URL] --sw 50-100  для общего стиля
```

Для V7 важно: старый --cref заменён на --oref (Omni Reference).
--oref лучше понимает ракурс камеры и сохраняет идентичность персонажа.

Параметр --ow (omni weight):
- 100-200: лёгкое влияние, больше свободы AI
- 300-400: баланс (рекомендуется для сцен)
- 500-700: сильное влияние (для крупных планов)
- 800-1000: почти копия (для character sheets)

Параметр --sw (style weight):
- 25-50: лёгкое стилевое влияние
- 50-100: нормальное (по умолчанию 100)
- 200-400: сильное стилевое влияние

**Уровень 3: Anchor Frame (якорный кадр)**
Выбрать 1 идеальный кадр из первого эпизода -- "якорь".
Все последующие кадры генерируются с этим якорем как --sref.

```
Файл: assets/references/anchor_frame.png
Критерии:
- Офис, 2-3 персонажа, тёплое освещение
- Идеальная цветовая температура
- Правильные пропорции персонажей
- Чистая композиция
```

### Workflow для серийного производства

```
1. Открыть промпт-шаблон серии
2. Вставить STYLE PREFIX
3. Написать описание сцены
4. Вставить STYLE SUFFIX
5. Добавить --oref [персонаж.png] --ow 400
6. Добавить --sref [anchor_frame.png] --sw 75
7. Генерировать 4 варианта
8. Выбрать лучший
9. Если дрифт -- увеличить --ow/--sw на 100
10. Если слишком "копия" -- уменьшить на 100
```

## 6. Реальные примеры и практики

### Из индустрии AI-анимации (2025-2026)

**Layer AI (production tool):**
Использует prefix/suffix систему: описание персонажа как prefix,
стиль как suffix. Автоматически склеивает для каждого промпта.
Это именно наш подход -- prefix=стиль, suffix=свет+атмосфера.

**Midjourney serial production:**
Профессиональные создатели AI-серий используют:
1. Master character sheet (наш PROMPTS.md)
2. Style reference image (наш anchor_frame.png)
3. Fixed prompt template (наш prefix+suffix)
4. Negative prompt library (наш Forbidden List)
5. Color palette reference (наш color_palette.png)

**Pixar production pipeline (адаптировано):**
Pixar создаёт "colour scripts" -- цветовую раскадровку ДО начала рендера.
Каждый кадр -- цветовой эскиз, привязанный к эмоции.
Мы адаптируем: в раскадровке скилла scenario добавлять поле MOOD/COLOR
к каждому кадру.

**Art Bible (стандарт индустрии):**
Студии анимации (Pixar, DreamWorks, и инди) создают Art Bible -- документ
арт-директора с визуальным стилем проекта. Обязательные разделы:
- Color palette
- Lighting rules
- Character proportions & silhouettes
- Environment/materials
- Forbidden elements
- Technical specs
Этот документ (ART_DIRECTION.md) -- наш Art Bible.

### Omni Reference -- ключевой инструмент 2026

В Midjourney V7 появился --oref (Omni Reference), заменивший --cref.
Преимущества:
- Лучше понимает ракурс камеры
- Сохраняет идентичность в разных позах
- Работает с объектами и локациями, не только лицами
- Поддерживает несколько референсов одновременно

Это значит: наши мастер-образы из PROMPTS.md теперь работают эффективнее.

## 11. Drift Detection -- чек-лист после генерации каждого кадра

### Что такое "визуальный дрифт"

Дрифт -- постепенное отклонение стиля от установленного стандарта. AI-модели не хранят
"память" о предыдущих генерациях. Каждый новый промпт -- генерация с нуля. Без якорей
(--oref, --sref, prompt DNA) стиль гарантированно уплывёт за 5-10 кадров.

Три типа дрифта:
1. **Character drift** -- меняются черты лица, пропорции, одежда, силуэт
2. **Style drift** -- меняется степень стилизации (то мультик, то фотореализм)
3. **Color drift** -- меняется цветовая температура, насыщенность, освещение

### Чек-лист: "5 секунд" (быстрая проверка глазами)

Открыть новый кадр рядом с anchor frame. За 5 секунд ответить:

| # | Вопрос | Да/Нет |
|---|--------|--------|
| 1 | Это из того же мультика? (общее ощущение стиля) | |
| 2 | Персонаж узнаваем по силуэту на расстоянии вытянутой руки? | |
| 3 | Цветовая температура тёплая? (нет холодных синих/серых тонов) | |
| 4 | Уровень детализации примерно одинаковый? (не hyperdetailed / не слишком простой) | |
| 5 | Это выглядит как кадр из НАШЕГО сериала, а не из другого? | |

**Если хоть один "Нет" -- не использовать. Регенерировать с усиленными якорями.**

### Чек-лист: детальная проверка (после прохождения "5 секунд")

**Персонаж:**
- [ ] Силуэт соответствует мастер-образу (пропорции тела, рост, комплекция)
- [ ] Цвет одежды точно соответствует цветовой карте (Hex-референс из раздела 2)
- [ ] Характерные атрибуты на месте (очки Клодища, берет Байрона, шаль Стеллы и т.д.)
- [ ] Лицо без деформаций (нос, глаза, рот пропорциональны)
- [ ] Руки нормальные (5 пальцев, правильные пропорции, нет сращений)
- [ ] Выражение лица читается и соответствует эмоции кадра
- [ ] Стилизация "Pixar-warm" -- не фотореалистичный, не аниме, не плоский

**Среда/фон:**
- [ ] Офис узнаваем как "наш" офис (современный, тёплый, Дубай за окном)
- [ ] Детали обжитости: кофе, бумаги, экраны, стикеры (не стерильный)
- [ ] Нет лишних элементов (текст, водяные знаки, логотипы, артефакты AI)
- [ ] Глубина резкости мягкая (фон чуть размыт)

**Свет и цвет:**
- [ ] Общая тональность тёплая (amber/golden/honey)
- [ ] Тени мягкие (нет резких чёрных теней)
- [ ] Тип освещения соответствует эмоции сцены (раздел 3)
- [ ] Цветовая температура не холоднее, чем требует сцена

**Техника:**
- [ ] Формат верный (9:16 для Reels или 16:9 для YouTube)
- [ ] Разрешение достаточное (Upscale сделан)
- [ ] Нет визуальных артефактов (размытые пятна, повторяющиеся текстуры)

## 12. Cross-Episode Continuity -- что неизменно, что может меняться

### НЕИЗМЕННЫЕ элементы (священные коровы)

Эти элементы НИКОГДА не меняются между сериями. Любое отклонение = брак.

| Элемент | Стандарт | Где зафиксирован |
|---------|----------|-----------------|
| Стиль рендера | stylized warm illustration, soft 3D, Pixar-style | Style Bible (раздел 1) |
| Пропорции персонажей | Мастер-образы (рост, комплекция, голова/тело) | `PROMPTS.md` + `masters/` |
| Цвет одежды персонажей | Цветовая карта с Hex | Раздел 2 |
| Характерные атрибуты | Очки Клодища, берет Байрона, шаль Стеллы... | `SERIES_BIBLE.md` |
| Офис -- базовая планировка | Открытый офис, стекло, Дубай за окном | CHARACTER_SHEETS.md (локации) |
| Тёплая тональность | Amber/golden/honey доминирует | Палитра (раздел 2) |
| Мягкие тени | Без резких контрастов в стандартных сценах | Раздел 3 |
| Юрий -- только реальное видео | Никогда не мультяшный | SERIES_BIBLE.md |
| Два мира не пересекаются | Юрий не в мульте, агенты не в реале | SERIES_BIBLE.md |

### ИЗМЕНЯЕМЫЕ элементы (допустимые вариации)

| Элемент | Что может меняться | Ограничения |
|---------|-------------------|-------------|
| Ракурс камеры | Любой (close-up, wide, OTS...) | Словарь из MIDJOURNEY_PROMPT_GUIDE |
| Поза персонажа | Любая в рамках характера | Силуэт остаётся узнаваемым |
| Выражение лица | Любое из словаря эмоций | Не гротескное, не аниме |
| Предметы в руках | Зависят от сцены | Атрибуты персонажа сохраняются |
| Детали фона | Разные зоны офиса | Стиль и палитра единые |
| Освещение | По эмоции сцены | Из словаря (раздел 3), в рамках палитры |
| Время суток | Утро/день/вечер/ночь | Цветовые правила по сценам (раздел 2) |
| Количество персонажей | 1-3 на кадр | При 3+ -- тестовая генерация |

### ОПАСНАЯ ЗОНА (меняется только с обоснованием)

| Элемент | Когда допустимо | Нужно одобрение Юрия |
|---------|----------------|---------------------|
| Локация (не офис) | Flashback, внешние сцены | Да |
| Одежда персонажа | Специальная серия (праздник, маскарад) | Да |
| Цветовая температура (холодная) | Конфликтная сцена, "мир сломался" | Да |
| Стиль рендера (отклонение) | Сон, видение, "глюк системы" | Да |

## 13. Color Consistency -- правила сохранения цвета

### Проблема

AI-модели не имеют абсолютного понятия цвета. "Purple hoodie" для Midjourney может быть
фиолетовым, сиреневым, лиловым или даже синим в зависимости от контекста промпта и seed.
Цвет -- главный маркер идентичности персонажа на маленьком экране телефона.

### Решение: тройной якорь цвета

**Якорь 1: Текстовый (в промпте)**
Использовать ТОЧНЫЕ цветовые слова из таблицы (раздел 2), не синонимы:
- "purple hoodie" -- ДА
- "violet sweatshirt" -- НЕТ (другой оттенок)
- "light blue shirt" -- ДА
- "sky blue top" -- НЕТ

**Якорь 2: Визуальный (--oref мастер-образ)**
Мастер-образ уже содержит правильный цвет. --oref с весом 300-500 передаёт цвет одежды.

**Якорь 3: Стилевой (--sref палитра или anchor frame)**
Anchor frame фиксирует общую цветовую температуру сцены.
Color palette как --sref (раздел 2) фиксирует диапазон допустимых цветов.

### Правила проверки цвета

1. **Eyedrop-тест:** Открыть новый кадр в любом редакторе (Preview на Mac, Photoshop, Figma).
   Пипеткой взять цвет одежды персонажа. Сравнить Hex с таблицей в разделе 2.
   Допустимое отклонение: +/- 15% по каждому каналу RGB. Больше = дрифт.

2. **Side-by-side тест:** Поставить новый кадр рядом с мастер-образом на одном экране.
   Цвет одежды должен читаться как "тот же" без подсказки.

3. **Thumbnail-тест:** Уменьшить кадр до 100x178px (размер превью в Instagram).
   Персонаж всё ещё идентифицируется по цвету? Если нет -- цвет уплыл.

### Частые проблемы и решения

| Проблема | Причина | Решение |
|----------|---------|---------|
| Цвет стал бледнее | Слово "soft" или "muted" в промпте | Убрать или заменить на "rich", "saturated" |
| Цвет стал ярче/кислотнее | "bright" или "vivid" в промпте | Убрать, использовать точное цветовое слово |
| Два персонажа -- похожий цвет | Midjourney усредняет | Генерировать раздельно, --oref для каждого |
| Фон "съедает" цвет персонажа | Фон слишком яркий или того же тона | Добавить контраст: "character standing out against..." |
| Тёплый свет делает всех "оранжевыми" | Слишком сильный golden lighting | Уменьшить: "warm" вместо "golden hour", или "balanced warm" |

## 14. QA-процесс -- кто, что и как проверяет

### Роли в QA

Для AI-генерированной анимации с маленькой командой (1-2 человека) роли совмещаются,
но проверки не пропускаются.

| Роль | Кто выполняет | Что проверяет |
|------|--------------|---------------|
| Art Director | Юрий | Общее ощущение, "наш/не наш", эмоция, узнаваемость |
| Style QA | Claude (сценарист) | Промпт соответствует Style Bible, нет запрещённых слов |
| Technical QA | Юрий | Разрешение, формат, артефакты, руки, лица |
| Color QA | Юрий | Eyedrop-тест, цвет одежды, температура |
| Continuity QA | Claude | Cross-episode проверка (раздел 12) |

### Workflow QA по этапам

```
ЭТАП 1: ПРОМПТ (до генерации)
├── Claude проверяет промпт по чек-листу раздела 8
├── Prefix/Suffix на месте?
├── Запрещённые слова отсутствуют?
├── --oref/--sref указаны?
└── Результат: промпт одобрен → генерация

ЭТАП 2: ГЕНЕРАЦИЯ (4 варианта от Midjourney)
├── Юрий делает "5-секундный тест" (раздел 11) по 4 вариантам
├── Выбирает лучший
├── Если ни один не подходит → корректировка промпта → повтор
└── Результат: выбран 1 вариант → Upscale

ЭТАП 3: UPSCALE (финальный кадр)
├── Детальная проверка (раздел 11, полный чек-лист)
├── Eyedrop-тест цвета (раздел 13)
├── Thumbnail-тест (раздел 13)
├── Сравнение с предыдущими кадрами серии (side-by-side)
└── Результат: кадр одобрен → в assets/scenes/

ЭТАП 4: СЕРИЯ ЦЕЛИКОМ (после генерации всех кадров)
├── Разложить все кадры серии в ряд (как color script)
├── Визуальная "прогулка": стиль ровный? цвет ровный? персонажи узнаваемы?
├── Сравнить с кадрами предыдущих серий (cross-episode)
├── Если есть выбивающийся кадр → регенерировать
└── Результат: серия одобрена → переход к img2vid / озвучке
```

### Правило "3 попытки"

Если кадр не проходит QA после 3 генераций:
1. Остановиться
2. Проанализировать что именно дрифтит
3. Усилить якоря (увеличить --ow/--sw, добавить --sref, упростить промпт)
4. Если не помогает -- изменить композицию кадра (возможно, запрошенная поза/ракурс
   конфликтует с сохранением идентичности)

## 15. Color Script -- эмоциональная карта серии

### Что это

Color script (цветовая раскадровка) -- техника Pixar. Последовательность маленьких
цветовых эскизов, по одному на каждый кадр серии. Показывает эмоциональную "температуру"
серии через цвет и свет.

Для AI Office: color script создаётся НА ЭТАПЕ РАСКАДРОВКИ (поле MOOD),
а проверяется ПОСЛЕ ГЕНЕРАЦИИ всех кадров.

### Шаблон color script для серии

```
СЕРИЯ: S01E__
ОБЩАЯ ТЕМПЕРАТУРА: тёплая (стандарт)
ЭМОЦИОНАЛЬНАЯ ДУГА: [спокойствие → проблема → поиск → инсайт → юмор]

КАДР 1 (зацепка):     ████ нейтрально-тёплый  | natural warm | Юрий спокоен
КАДР 2 (переход):     ████ нейтральный         | screen glow  | ныряем в мульт
КАДР 3 (ситуация):    ████ тёплый              | golden hour  | офис, обычный день
КАДР 4 (проблема):    ████ приглушённо-тёплый  | muted warm   | что-то не так
КАДР 5 (поворот):     ████ контрастный тёплый  | rim light    | момент истины
КАДР 6 (решение):     ████ яркий тёплый        | bright warm  | инсайт!
КАДР 7 (переход):     ████ нейтральный         | screen glow  | обратно в реал
КАДР 8 (финал):       ████ тёплый + мягкий     | golden soft  | Юрий подводит итог
```

### Правила color script

1. Серия НАЧИНАЕТСЯ и ЗАКАНЧИВАЕТСЯ в одной температуре (замкнутая дуга)
2. Максимум 1 "холодный" момент на серию (конфликт/проблема)
3. Финальный кадр -- всегда теплее или равен первому (зритель уходит с хорошим чувством)
4. Переходы реал/мульт -- нейтральные (не тёплые, не холодные)
5. Общее правило: 70% кадров -- стандартный тёплый, 20% -- вариации, 10% -- контраст

## 16. Anchor System -- якорная система для 20+ эпизодов

### Проблема масштаба

Для 1-2 серий достаточно одного anchor frame. Для 20+ серий нужна иерархия якорей,
иначе стиль постепенно уплывёт (каждая серия чуть дрифтит от предыдущей,
и серия 20 уже не похожа на серию 1).

### Трёхуровневая якорная система

```
УРОВЕНЬ 1: МАСТЕР-ЯКОРЬ (один на весь сериал)
│  Файл: assets/references/master_anchor.png
│  Что: идеальный кадр с 2-3 персонажами в офисе
│  Используется: --sref для КАЖДОГО кадра КАЖДОЙ серии
│  Меняется: НИКОГДА
│
├── УРОВЕНЬ 2: СЕЗОННЫЙ ЯКОРЬ (один на сезон)
│   │  Файл: assets/references/season_01_anchor.png
│   │  Что: лучший кадр первой серии сезона
│   │  Используется: дополнительный --sref при заметном дрифте
│   │  Меняется: раз в сезон
│   │
│   └── УРОВЕНЬ 3: ЭПИЗОДНЫЙ ЯКОРЬ (один на серию)
│       Файл: assets/scenes/S01EXX/anchor.png
│       Что: первый одобренный кадр серии
│       Используется: side-by-side сравнение для остальных кадров серии
│       Меняется: каждую серию
```

### Выбор мастер-якоря (критерии)

Мастер-якорь -- самое важное изображение проекта. Он определяет стиль на 20+ серий.

- [ ] 2-3 персонажа в кадре (не один -- нужен контекст масштаба)
- [ ] Офис AI Office с деталями (стол, экран, кофе, Дубай за окном)
- [ ] Идеальная цветовая температура (тёплая, amber/golden)
- [ ] Мягкие тени, depth of field
- [ ] Пропорции персонажей правильные
- [ ] Стиль = 100% "stylized warm illustration, Pixar-style"
- [ ] Нет артефактов, нет текста, нет водяных знаков
- [ ] При уменьшении до 200px персонажи всё ещё читаются

### Ротация и обновление якорей

| Событие | Действие |
|---------|----------|
| Начало сериала | Создать мастер-якорь + сезонный якорь |
| Каждая серия | Выбрать эпизодный якорь (первый одобренный кадр) |
| Начало нового сезона | Создать новый сезонный якорь |
| Обновление Midjourney (новая версия) | Пересоздать мастер-якорь в новой версии, сравнить |
| Заметный дрифт | Усилить --sw на 25-50, добавить второй --sref |

## 17. Инструменты проверки

### Бесплатные инструменты для визуального QA

| Инструмент | Что делает | Когда использовать |
|-----------|-----------|-------------------|
| **Preview (macOS)** | Пипетка цвета (Hex), side-by-side | Eyedrop-тест, сравнение 2 кадров |
| **Figma (бесплатный план)** | Разложить все кадры серии в ряд | Color script проверка, обзор серии |
| **XKit Image Compare** (xkit.io/difftools) | Наложение двух изображений, diff | Сравнение нового кадра с anchor |
| **ODiff** (github.com/dmtrKovalenko/odiff) | Быстрое pixel-level сравнение | Автоматизация проверки дрифта |
| **DigitalColorMeter** (macOS) | Пипетка цвета в любом приложении | Быстрая проверка Hex без открытия редактора |

### Метод "Стена кадров" (самый простой и эффективный)

1. Открыть Figma / Keynote / Preview
2. Разложить ВСЕ кадры серии горизонтально в порядке раскадровки
3. Уменьшить масштаб до 50-70%
4. Пробежать глазами слева направо

**Что искать:**
- Кадр, который "выбивается" из ряда (другой стиль, температура, детализация)
- Скачок яркости/насыщенности между соседними кадрами
- Персонаж, который выглядит "не как обычно"

**Для cross-episode проверки:** добавить в ту же стену кадры из предыдущей серии.
Стиль должен быть неразличим.

### Метод "Blur test"

1. Размыть кадр гауссовым блюром (radius 10-15px)
2. Остаются только крупные цветовые пятна
3. Цветовые пятна должны соответствовать палитре серии
4. Если появляются чужеродные цвета -- кадр уплыл

## 18. Style Guide Template -- полная структура документа арт-дирекции

### Для чего этот раздел

```
ART BIBLE / ART DIRECTION DOCUMENT

6. ENVIRONMENT DESIGN (1-2 страницы на локацию)
   - Базовая планировка

7. CAMERA & COMPOSITION (1 страница)
   - Разрешённые ракурсы (словарь)

   - Правило третей / композиционные принципы

8. CONSISTENCY SYSTEM (1-2 страницы)
   - Якорная система (master → season → episode anchors)

   - Workflow генерации серии

9. QA PROCESS (1-2 страницы)
   - Быстрый чек-лист ("5 секунд")
   - Детальный чек-лист
   - Инструменты проверки
   - Роли и ответственности
   - Правило "3 попытки"

10. TECHNICAL SPECS (1 страница)
    - Разрешение (генерация и финальное)
    - Формат файлов
    - Именование файлов
    - Версия AI-модели (зафиксировать!)
    - Структура папок

ПРИЛОЖЕНИЯ:
A. Quick Reference Card (1 страница, для печати)
B. Color Palette (визуальная, для --sref)
C. Мастер-образы всех персонажей
D. Anchor frames
E. Примеры "правильно / неправильно"
```

## 19. Практические примеры из индустрии

### Pixar: Color Scripts

Pixar создаёт цветовые раскадровки (color scripts) ДО начала рендера каждого фильма.
Каждый кадр -- миниатюра, привязанная к эмоции через цвет. Тёплые тона = комфорт,
холодные = конфликт. "The Art of Pixar: The Complete Colorscripts" -- книга-референс.

**Что берём:** Поле MOOD в раскадровке + color script (раздел 15).

### Анимационные студии: Limited Wardrobe

95% анимационных сериалов используют "limited wardrobe" -- персонажи носят одну и ту же
одежду во всех сериях. Причина: каждая смена одежды = новый дизайн + утверждение +
отрисовка. Для AI-генерации: смена одежды = новый мастер-образ + тестирование
консистентности = дорого по времени и GPU.

**Что берём:** Персонажи AI Office не меняют одежду. Исключения -- с одобрения Юрия.

### Fruit Planet (2025): 5 человек, 2 недели, AI

Продакшен-команда из 5 человек создала первый эпизод AI-анимации за 2 недели.
Метод: генерация character designs → GigaPixel upscale → ручная доработка
глаз и ртов в Photoshop (AI плохо генерирует выражения для lip sync).

**Что берём:** Готовность к ручной доработке глаз/ртов если нужен lip sync.

### Midjourney Serial Creators (2025-2026 community)

Профессиональные создатели AI-серий используют 5 якорей одновременно:
1. Master character sheet (наш PROMPTS.md)
2. Style reference image (наш anchor_frame)
3. Fixed prompt template (наш prefix+suffix)
4. Negative prompt library (наш Forbidden List)
5. Seed locking (фиксация seed для серии)

**Что берём:** Seed locking -- записывать seed удачных генераций, использовать --seed
для кадров с тем же персонажем в той же серии.

### Midjourney V8 Alpha (март 2026)

V8 улучшает консистентность персонажей между генерациями. Точнее следует сложным
промптам (палитры, расположение, освещение, текстуры). Лучше детализирует глаза
и мелкие элементы. Стоит протестировать для AI Office когда выйдет из альфы.

**Что берём:** Следить за V8, тестировать параллельно с V7/V6.1.
