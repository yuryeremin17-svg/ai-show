## 1. Kurzgesagt (In a Nutshell) — золотой стандарт edutainment-анимации

**Процесс (1200+ часов на видео):**

1. **Глубокий ресёрч** — книги, первичные источники, научные статьи. Всегда консультация с экспертами в теме. Факт-чекинг и составление source sheet занимают ~100 часов на видео.
2. **Скрипт** — сценарист формирует историю и угол подачи. Работает в тандеме с ресёрчером. Около **12 драфтов** — отсекают лишнее, полируют баланс между фактами и историей.
3. **Факт-чекинг** — 2-3 человека проверяют скрипт в глубину, плюс 1-3 внешних эксперта дают фидбек.
4. **Раскадровка** — покадровый план, визуальные метафоры, переходы.
5. **Озвучка** — Steve Taylor перезаписывает абзацы многократно, пока пейсинг идеален.
6. **Анимация** — 2-3 аниматора, 8-12 недель. ~200 уникальных иллюстраций на 10-минутное видео.

**Сценарные приёмы Kurzgesagt:**

| Приём | Как работает |
|-------|-------------|
| **Одна яркая метафора** | Не 5 быстрых примеров, а один сценарий — и 90-120 секунд его раскрытия через разные аспекты. Зритель строит цельную ментальную модель |
| **Визуальные метафоры** | Абстрактные идеи привязаны к знакомым объектам (дворецкий, водитель/пассажир). Метафора видна на картинке без текста |
| **Баланс точность/история** | ~12 драфтов — каждый ищет грань между "правильно" и "увлекательно" |
| **Перспективный сдвиг** | Начинают с одного масштаба, переходят на другой (от атома к вселенной, от личного к глобальному) |
| **Уникальный саундтрек** | Каждое видео — своя музыкальная тема для эмоционального воздействия |

**Что взять для AI_SHOW:**
- Принцип "одна метафора = одна серия" (у нас уже есть "один инсайт")
- 12 драфтов — перебор для 60 сек, но 3-4 драфта — must
- Факт-чекинг по MY_STORIES.md — аналог их source sheet
- Визуальная метафора должна работать БЕЗ текста (тест "без звука")

## 2. TED-Ed — структура образовательного анимированного урока

**Процесс:**
1. Педагог формулирует **learning objective** — одно предложение: что зритель поймёт к концу видео
2. Скрипт пишется в **двухколоночном формате** (тайм-коды + действие/текст)
3. Первые 30 секунд — захватить внимание
4. Ключевое сообщение доставляется в основной части
5. Заключение — закрепление + следующий шаг

**Правила TED-Ed:**
- 125-150 слов = 1 минута озвучки
- Язык максимально простой — как если объясняешь знакомому, который никогда не слышал о теме
- Цель видео определяет всё: стиль, детализацию, язык
- Скрипт утверждается ДО начала анимации (checkpoint "script approval")

**Что взять для AI_SHOW:**
- Learning objective = наш "инсайт серии" в одном предложении. Уже есть.
- Формат двух колонок (аудио + визуал) — у нас уже в раскадровке, но можно формализовать
- Правило 125-150 слов/минуту (для русского ~100-130 — у нас уже в SKILL.md)

## 3. Смешарики — как работает российская анимация

**Факты о процессе:**
- Сценарий одной серии = **4-5 страниц**
- Хронометраж серии: 6 минут 30 секунд
- Сценаристы: Дима Яковенко и Светлана Мардаголимова
- Персонажей изначально было 20, сократили до 9 (Салават Шайхинуров)

**Процесс:**
1. Сценаристы пишут идею и текст
2. Авторы правят, отправляют режиссёрам
3. Озвучка актёрами
4. Раскадровка
5. **Аниматик** = раскадровка + озвучка (ещё не мультфильм, но уже "комикс со звуком")
6. Анимация (16-18 рисунков = 1 секунда)
7. Совмещение видео + аудио дорожек

**Что взять для AI_SHOW:**
- Аниматик как промежуточный шаг — у нас это по сути "голос + картинки перед сборкой в видео"
- Сокращение персонажей до необходимого (у нас 14 — это много для 60-секундных серий, в каждой серии 2-3 агента максимум)
- Формула "4-5 страниц на 6.5 минут" = ~0.7 страницы/минута. Для 60 сек = ~0.7 страницы. Наша раскадровка примерно столько и есть.

## 4. Структура 60-секундного Reels / TikTok (русскоязычные практики)

### Трёхактная структура для 60 секунд

```
АКТ 1 — ЗАВЯЗКА (0:01-0:15)
  Мощный хук. Мгновенно обозначить проблему/тему.
  Пример: "Вы до сих пор делаете X? Это огромная ошибка!"

АКТ 2 — КУЛЬМИНАЦИЯ (0:16-0:50)
  Решение, лайфхак, процесс. Раскрыть основную мысль.
  Здесь вся ценность ролика.

АКТ 3 — РАЗВЯЗКА (0:51-0:60)
  Призыв к действию. Инсайт. Вопрос.
  "Сохрани / поделись / а как у тебя?"
```

### Альтернативные формулы

| Формула | Структура | Когда работает |
|---------|----------|----------------|
| **AIDA** | Внимание → Интерес → Желание → Действие | Продающий контент |
| **HPS** | Хук → Польза/история → Следующий шаг | Образовательный контент |
| **3 акта** | Завязка → Развитие → Развязка | Сторителлинг |
| **Боль-Решение** | Боль → Решение → Результат → Призыв | Экспертный контент |
| **Хук-Инсайт-Призыв** | Хук → Инсайт → Призыв | Быстрый формат 30 сек |

### Правило первых 3 секунд

> 70%+ вирусных видео имеют сильный хук в первые 3 секунды. Если нет хука — зритель свайпает.

**Типы хуков:**
- Вопрос ("Знаешь зачем AI-агенту кофе?")
- Боль ("Презентация для клиента. Luxury Дубай. А выглядит — колхоз.")
- Шок-факт ("Мой AI подписал отчёт не читая.")
- Обещание ("За 60 секунд покажу как AI сломал мне проект")
- Действие с середины (начинаем в момент конфликта, контекст даём потом)

### Правило глубины просмотра

Ключевой показатель для алгоритмов Instagram/TikTok — **retention** (досмотр до конца). Сценарий должен удерживать каждую секунду. Если зритель проматывает — алгоритм хоронит видео.

**Что взять для AI_SHOW:**
- Наша структура "два мира" УЖЕ включает трёхактную схему (зацепка → мульт → инсайт)
- Хук в первые 3 секунды — критично. Юрий должен зацепить С ПЕРВОЙ ФРАЗЫ
- Формула "Хук → Инсайт → Вопрос" идеально ложится на наш формат

## 5. Explainer video — промышленный стандарт

### Двухколоночный AV-скрипт (индустриальный стандарт)

```
| ВРЕМЯ  | ВИЗУАЛ (Video)                    | АУДИО (Audio)                     |
|--------|-----------------------------------|-----------------------------------|
| 0:00   | Камера на Юрия, ноутбук открыт   | ЮРИЙ: "Презентация для клиента..." |
| 0:05   | Крупный план экрана               | SFX: клик мыши                     |
| 0:10   | Переход: камера ныряет в экран   | МУЗЫКА: нарастание                 |
| ...    | ...                               | ...                               |
```

### Формула структуры (Breadnbeyond Cheatsheet)

```
Проблема (30%) → Решение (40%) → Доказательство (20%) → Призыв (10%)
```

Для 60-секундного видео:
- 0:00-0:18: Проблема
- 0:18-0:42: Решение
- 0:42-0:54: Доказательство/результат
- 0:54-1:00: Призыв/инсайт

### Правила скрипта для explainer video

**Длина:**
- 60 секунд = 150-160 слов (англ.) / 100-130 слов (рус.)
- 90 секунд = 200-250 слов
- Максимальная длина предложения: 20 слов

**DO:**
- Двухколоночный формат (аудио | визуал)
- Первые 7-8 секунд = хук
- Говори как другу, который никогда не слышал о теме
- Прочитай скрипт вслух — ловишь кривые фразы
- Обязательный чекпоинт "script approval" ДО анимации (дешевле править текст, чем перерисовывать)
- "Meet Bob" формула: дай зрителю протагониста с теми же проблемами

**DON'T:**
- Не пытайся сказать всё за 60 секунд
- Не используй жаргон и длинные предложения
- Не уходи от темы
- Один набор переходов на всё видео (если плавные — все плавные, если резкие — все резкие)

**Что взять для AI_SHOW:**
- Правило 20 слов/предложение — хороший лимит для озвучки
- "Meet Bob" = "Meet Юрий" — у нас это уже заложено в формат
- Script approval ДО генерации картинок — сэкономит кучу времени
- Один набор переходов — у нас уже: Seedance dive-in / zoom-out

## 6. Pixar — 22 правила сторителлинга (Эмма Коутс)

Полные 22 правила опубликованы бывшей раскадровщицей Pixar. Самые релевантные для AI_SHOW:

| # | Правило | Применение к AI_SHOW |
|---|---------|---------------------|
| 1 | Героя ценят за старания, не за успехи | Юрий пробует, ошибается, пробует снова |
| 2 | Помни что интересно ЗРИТЕЛЮ, не автору | Не "как я настроил API", а "что из этого вышло" |
| 4 | Формула: "Жил-был ___. Каждый день ___. Однажды ___. Из-за этого ___. Пока наконец ___." | Идеальный скелет для 60 секунд |
| 5 | Упрощай. Сфокусируйся. Комбинируй персонажей. Пропускай обходные пути | В серии — 2-3 агента, не 14 |
| 6 | В чём твой герой хорош? Дай ему противоположное | Юрий = опыт управления людьми. Противоположное = AI, который не ведёт себя как люди |
| 7 | Придумай финал ДО середины | Инсайт серии определяется первым |
| 9 | Когда застрял — составь список того, что НЕ произойдёт. Удивишься | Инструмент для сценариста |
| 11 | Записать на бумагу — значит начать чинить. Идея в голове = идеальная и непроверенная | Первый драфт — быстро, не идеально |
| 16 | Что на кону? Почему зрителю сопереживать? Что будет если герой провалится? | Ставки: клиент, презентация, репутация |
| 19 | Совпадения, которые загоняют героя в беду — хорошо. Совпадения, которые спасают — лениво | Не "AI вдруг всё починил". А "AI наломал дров, Юрий нашёл выход" |
| 22 | Какова суть истории МАКСИМАЛЬНО КОРОТКО? Когда знаешь — можешь начинать | = наш "инсайт серии" в одном предложении |

**Формула Pixar (правило #4) адаптированная для AI_SHOW:**
```
Жил-был [Юрий], который каждый день [управлял AI-офисом].
Однажды [конкретная ситуация из MY_STORIES].
Из-за этого [конфликт/проблема].
Из-за этого [эскалация].
Пока наконец [инсайт/решение].
```

## 7. Правила анимационного сценария (из индустрии)

### Из seance.ru (журнал "Сеанс" о кино)

1. **Локации = эмоции.** Описание места должно вызывать чувства, даже в нескольких строчках
2. **Монтажное мышление.** Сценарист должен "видеть" кадры: крупный план, общий план, наезд, отъезд. Пример из Пушкина "Полтава" — классика монтажной прозы
3. **Итеративный процесс.** На Dreamworks висит плакат: "Пишем сценарий → раскадровка → аниматик → СНОВА переписываем сценарий"
4. **Максимальная выразительность.** Анимация не имеет права быть "недостаточно выразительной" — каждый жест, каждое действие отточены

### Из vc.ru (практика 2D-анимации)

1. **Один набор переходов** на весь ролик (все плавные ИЛИ все резкие — не смешивать)
2. **Техническая реализуемость.** Если не можешь представить как это анимировать — перепиши сцену
3. **Рисуй раскадровку.** Если не можешь нарисовать даже примитивно — сценарий недоработан
4. **Термины аниматоров.** Описания должны быть однозначными, с единой терминологией

## 8. Формулы хуков для Short-Form видео

### 7 фреймворков вирусных хуков (CreatorsJet)

1. **"Вот как..."** — "Here's how to fix [боль] in 60 seconds" → обещание быстрого инсайта
2. **Середина действия** — начать с кульминационного момента, контекст потом
3. **Противоречие** — "AI-агент отказался работать. И был прав."
4. **Вопрос** — "Как вы думаете, сколько раз AI переспросил одно и то же?"
5. **Шок-факт** — "Мой AI подписал отчёт, не читая. Как мой зам в 2015."
6. **Обещание списка** — "3 ошибки которые делают все с AI-ассистентами"
7. **Личная история** — "Вчера мой AI чуть не потерял клиента..."

### Правило 8 секунд

> Первые 8 секунд определяют, будет ли зритель смотреть дальше. Для видео 45-60 секунд хук должен обещать достаточно ценности, чтобы оправдать время.

## 9. ИТОГОВЫЙ ЧЕКЛИСТ для сценариста AI_SHOW

### Перед написанием

- [ ] Инсайт серии сформулирован в ОДНОМ предложении
- [ ] История из MY_STORIES.md прочитана, факты выписаны
- [ ] Определены 2-3 персонажа для этой серии (не больше)
- [ ] Хук придуман (какой тип: вопрос / боль / шок / действие?)
- [ ] Формула Pixar заполнена: "Жил-был ___, однажды ___, из-за этого ___..."

### Во время написания

- [ ] Первые 3 секунды — хук, без "привет, сегодня расскажу..."
- [ ] Каждое предложение <= 20 слов
- [ ] Весь текст озвучки <= 130 слов (для 60 секунд, русский)
- [ ] Каждый кадр = одна мысль + одна картинка
- [ ] Эмоция зрителя меняется каждые 7-10 секунд
- [ ] Не три кадра подряд с одинаковым ритмом
- [ ] Визуальная метафора работает БЕЗ текста (тест без звука)
- [ ] Юрий только в реальных шотах, агенты только в мульте
- [ ] Переходы между мирами = отдельные шоты (не эффект монтажа)
- [ ] Один набор переходов на всё видео
